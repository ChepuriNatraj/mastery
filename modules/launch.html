Launch Files

Orchestrate multi-node systems with remaps, namespaces, and parameters.

CPU-Light Required

What / Why / When

What

Launch files start multiple nodes with coordinated configuration in a single command. ROS2 supports Python launch files (programmatic) and YAML launch files (declarative). They handle node startup, parameter loading, topic remapping, namespace assignment, and conditional execution. Launch files can include other launch files and pass arguments between them.

Why

Starting complex systems node-by-node in separate terminals is error-prone and slow. Launch files ensure reproducible startup sequences, centralize configuration, and enable environment-specific setups. They're essential for production systems where dozens of nodes must start with correct relationships. Launch arguments allow single-source configurations for different robots or environments.

When

Use launch files when running more than one node together. For development, Python launch provides flexibility with conditionals and computed values. For simple multi-node systems, YAML launch is cleaner. Use composable nodes in launch files to reduce process overhead. Always version-control launch files alongside code for deployment reproducibility.

10-Minute Drill

Python Launch (C++ ecosystem)

Create a launch file starting talker and listener with parameters:

# launch/talker_listener.launch.py
from launch import LaunchDescription
from launch_ros.actions import Node
from launch.actions import DeclareLaunchArgument
from launch.substitutions import LaunchConfiguration

def generate_launch_description():
    # Declare launch argument
    use_sim_time_arg = DeclareLaunchArgument(
        'use_sim_time',
        default_value='____',
        description='Use simulation time'
    )
    
    # Get launch configuration
    use_sim_time = LaunchConfiguration('____')
    
    return LaunchDescription([
        use_sim_time_arg,
        
        Node(
            package='my_pkg',
            executable='____',
            name='talker',
            output='screen',
            parameters=[{'use_sim_time': ____}]
        ),
        
        Node(
            package='my_pkg',
            executable='____',
            name='listener',
            output='screen',
            parameters=[{'use_sim_time': use_sim_time}],
            remappings=[
                ('chatter', '____')  # Remap to topic_name
            ]
        )
    ])

YAML Launch (Python ecosystem)

Equivalent declarative launch file:

# launch/talker_listener.launch.yaml
launch:
  - arg:
      name: use_sim_time
      default: "false"
      description: "Use simulation time"
  
  - node:
      pkg: my_pkg
      exec: ____
      name: talker
      output: screen
      param:
        - name: use_sim_time
          value: $(var use_sim_time)
  
  - node:
      pkg: my_pkg
      exec: ____
      name: listener
      output: screen
      param:
        - name: use_sim_time
          value: $(var use_sim_time)
      remap:
        - from: chatter
          to: ____

Advanced Launch with Conditionals

from launch.conditions import IfCondition, UnlessCondition
from launch.substitutions import LaunchConfiguration

def generate_launch_description():
    use_rviz = LaunchConfiguration('use_rviz')
    
    return LaunchDescription([
        DeclareLaunchArgument('use_rviz', default_value='true'),
        
        # Launch RViz only if use_rviz is true
        Node(
            package='rviz2',
            executable='rviz2',
            name='rviz2',
            condition=IfCondition(use_rviz),
            arguments=['-d', rviz_config_path]
        ),
        
        # Launch headless node unless RViz is used
        Node(
            package='my_pkg',
            executable='headless_node',
            condition=UnlessCondition(use_rviz)
        )
    ])

Include Other Launch Files

from launch.actions import IncludeLaunchDescription
from launch.launch_description_sources import PythonLaunchDescriptionSource
from ament_index_python.packages import get_package_share_directory
import os

def generate_launch_description():
    robot_description_launch = IncludeLaunchDescription(
        PythonLaunchDescriptionSource([
            os.path.join(
                get_package_share_directory('robot_description'),
                'launch',
                'robot_state_publisher.launch.py'
            )
        ]),
        launch_arguments={
            'use_sim_time': 'true',
            'robot_name': 'turtlebot3'
        }.items()
    )
    
    return LaunchDescription([robot_description_launch])

Composable Nodes (Reduce Overhead)

from launch_ros.actions import ComposableNodeContainer
from launch_ros.descriptions import ComposableNode

def generate_launch_description():
    container = ComposableNodeContainer(
        name='my_container',
        namespace='',
        package='rclcpp_components',
        executable='component_container',
        composable_node_descriptions=[
            ComposableNode(
                package='my_pkg',
                plugin='my_pkg::TalkerComponent',
                name='talker'
            ),
            ComposableNode(
                package='my_pkg',
                plugin='my_pkg::ListenerComponent',
                name='listener'
            )
        ],
        output='screen'
    )
    
    return LaunchDescription([container])

Event Handlers and Actions

from launch.actions import RegisterEventHandler, LogInfo
from launch.event_handlers import OnProcessExit

def generate_launch_description():
    talker_node = Node(
        package='my_pkg',
        executable='talker',
        name='talker'
    )
    
    # Action when talker exits
    exit_handler = RegisterEventHandler(
        OnProcessExit(
            target_action=talker_node,
            on_exit=[
                LogInfo(msg='Talker exited, restarting...'),
                talker_node  # Restart node
            ]
        )
    )
    
    return LaunchDescription([talker_node, exit_handler])

Verify

Command-Line Checks

# Launch with default arguments
ros2 launch my_pkg talker_listener.launch.py

# Override launch arguments
ros2 launch my_pkg talker_listener.launch.py use_sim_time:=true

# Show available launch arguments
ros2 launch my_pkg talker_listener.launch.py --show-args
# Expected: Arguments (pass arguments as '<name>:=<value>'):
#   'use_sim_time': Use simulation time (default: 'false')

# List running nodes
ros2 node list
# Expected: /talker, /listener

# Check node parameters
ros2 param list /talker
# Expected: use_sim_time, ...

# Verify remapping
ros2 topic list
# Expected: /topic_name (remapped from /chatter)

Debug Launch Files

# Print launch description without executing
ros2 launch my_pkg talker_listener.launch.py --print-description

# Show launch file source
ros2 launch my_pkg talker_listener.launch.py --print

# Verbose output for debugging
ros2 launch my_pkg talker_listener.launch.py --debug
Pro Tip:

Use output='screen' for all nodes during development to see logs. For production, use output='log' and monitor with ros2 node info.

Common Failures & Fix Patterns

Failure 1: Package Not Found

Symptom: Package 'my_pkg' not found error when launching.

Root Cause: Package not built or workspace not sourced before launch.

Fix: Build and source workspace:

cd ~/ros2_ws
colcon build --packages-select my_pkg
source install/setup.bash
ros2 launch my_pkg my_launch.launch.py

Failure 2: Parameter File Not Found

Symptom: Failed to load parameter file at launch.

Root Cause: Parameter file path incorrect or not installed to share directory.

Fix: Use proper path resolution and install files:

# In launch file
from ament_index_python.packages import get_package_share_directory
import os

config_file = os.path.join(
    get_package_share_directory('my_pkg'),
    'config',
    'params.yaml'
)

# In setup.py or CMakeLists.txt, install config files
# setup.py:
data_files=[
    ('share/' + package_name + '/config', ['config/params.yaml']),
]

# CMakeLists.txt:
install(DIRECTORY config
  DESTINATION share/${PROJECT_NAME}
)

Failure 3: Name Collision

Symptom: Nodes fail to start with "node name already exists" error.

Root Cause: Multiple nodes launched with same name in same namespace.

Fix: Use unique names or namespaces:

# Different names
Node(package='my_pkg', executable='node', name='node_1')
Node(package='my_pkg', executable='node', name='node_2')

# Or different namespaces
Node(package='my_pkg', executable='node', namespace='robot1')
Node(package='my_pkg', executable='node', namespace='robot2')

# Or use anonymous names (adds unique suffix)
Node(package='my_pkg', executable='node', name='node', 
     ros_arguments=['--ros-args', '--enable-rosout-logs'])

Failure Injection Drill

Exercise: Invalid Argument Handling

Create launch file with argument validation:

from launch import LaunchDescription
from launch.actions import DeclareLaunchArgument, OpaqueFunction
from launch.substitutions import LaunchConfiguration

def validate_and_launch(context):
    robot_name = LaunchConfiguration('robot_name').perform(context)
    
    valid_robots = ['turtle1', 'turtle2', 'turtle3']
    if robot_name not in valid_robots:
        raise ValueError(f'Invalid robot_name: {robot_name}. '
                        f'Must be one of {valid_robots}')
    
    return [
        Node(
            package='my_pkg',
            executable='robot_node',
            name=robot_name
        )
    ]

def generate_launch_description():
    return LaunchDescription([
        DeclareLaunchArgument('robot_name', default_value='turtle1'),
        OpaqueFunction(function=validate_and_launch)
    ])

Detection: Launch with invalid argument:

ros2 launch my_pkg robot.launch.py robot_name:=invalid_bot
# Expected: ValueError with clear message about valid options

Recovery: Provide helpful error messages listing valid choices.

Warning:

Launch files don't have type checking by default. Always validate arguments and file paths early in launch execution to fail fast with clear messages.
