MoveIt2 Deep Dive

Plan collision-free arm motions, solve inverse kinematics, compute Cartesian paths, and integrate grippers with MoveIt2.

MoveIt2 Motion Planning Manipulation

What / Why / When

What

MoveIt2 is the primary motion-planning framework for ROS2. It wraps OMPL (Open Motion Planning Library) and exposes a unified C++ and Python API. Core concepts: PlanningScene (3D collision world with robot + obstacles), MoveGroupInterface (high-level planning + execution), RobotModel/RobotState (URDF-parsed kinematics chain), and TrajectoryExecution (send joint trajectories to controllers). MoveIt2 supports multiple IK solvers, multiple motion planners, and plugins for grasp generation, perception, and servo control.

Why

Arm motion is far more complex than mobile navigation: 6-7 DOF joint spaces, self-collision avoidance, workspace limits, and tool orientation constraints. OMPL planners search high-dimensional configuration spaces that A* cannot handle. IK solvers translate Cartesian end-effector goals into joint angles. Cartesian path APIs let you move the tool tip in a straight line — essential for welding, painting, and pick-and-place. Without MoveIt2, you would need to implement all of this from scratch.

When

Use MoveIt2 whenever you have a robotic arm (manipulator) that needs collision-aware motion planning. Use RRT-Connect for fast joint-space planning in unconstrained spaces. Use PRM when you need to plan many queries in the same static environment. Use TRAC-IK instead of KDL when the default IK solver fails to find solutions near workspace limits. Use Cartesian planning for straight-line tool paths. Integrate gripper actions after the arm reaches the pre-grasp pose.

OMPL Planner Comparison

Planner Algorithm Family Speed Path Quality Best Use Case
RRTConnect Sampling (bi-directional RRT) Very Fast Unoptimized, jagged Single-query, unconstrained spaces
RRTstar Sampling (asymptotically optimal) Slow Asymptotically optimal When path length/quality matters
PRM Probabilistic Roadmap Slow build, fast query Good after roadmap built Multi-query, static environments
PRMstar PRM (asymptotically optimal) Slower build Near-optimal High-quality multi-query planning
EST Expansive-Space Trees Medium Medium Narrow passages, constrained spaces
LBTRRT Lower-Bound Tree RRT Fast Near-optimal fast Real-time replanning

RRT vs PRM — When to Use Which

RRT (Rapidly-exploring Random Tree)
────────────────────────────────────
Idea:    Grow a tree from start by sampling random configs.
         Extend nearest node toward sample. Stop when goal reached.
Pros:    Single-query fast, handles any constraint
Cons:    Non-optimal path, must re-plan every query
Use:     Robot arm needs to reach a new pose quickly (pick & place)

RRT-Connect (bidirectional):
  - Grows TWO trees: one from start, one from goal
  - Connects them when close enough
  - ~10x faster than single-direction RRT
  - DEFAULT choice in MoveIt2

PRM (Probabilistic Roadmap Method)
────────────────────────────────────
Phase 1: Build roadmap by sampling N random collision-free configs.
         Connect nearby configs with local planner edges.
Phase 2: Connect start + goal to roadmap, query shortest path.
Pros:    Amortises build cost over many queries — fast at query time
Cons:    Expensive upfront; bad for dynamic environments
Use:     Assembly lines — same robot, same cell, hundreds of plans/day

Decision rule:
  Single goal, one-shot query  →  RRT-Connect
  Same environment, many goals →  PRM
  Path quality critical        →  RRT* or PRM*
  Narrow passages              →  EST or BKPIECE

MoveGroupInterface — Plan & Execute (C++)

// arm_mover.cpp
#include <rclcpp/rclcpp.hpp>
#include <moveit/move_group_interface/move_group_interface.h>
#include <moveit/planning_scene_interface/planning_scene_interface.h>
#include <geometry_msgs/msg/pose.hpp>

int main(int argc, char** argv) {
  rclcpp::init(argc, argv);
  auto node = std::make_shared<rclcpp::Node>(
    "arm_mover",
    rclcpp::NodeOptions().automatically_declare_parameters_from_overrides(true));

  // Spin in background thread (MoveIt2 requires this)
  rclcpp::executors::SingleThreadedExecutor executor;
  executor.add_node(node);
  auto spinner = std::thread([&executor]() { executor.spin(); });

  // Create MoveGroupInterface for "arm" planning group
  moveit::planning_interface::MoveGroupInterface move_group(node, "arm");
  move_group.setPlanningTime(5.0);    // Max 5 seconds to find plan
  move_group.setNumPlanningAttempts(3);

  // --- Joint-space goal ---
  std::vector<double> joint_goal = {0.0, -1.57, 0.0, -1.57, 0.0, 0.0};
  move_group.setJointValueTarget(____);  // joint_goal

  moveit::planning_interface::MoveGroupInterface::Plan plan;
  bool success = (move_group.plan(plan) ==
                  moveit::core::MoveItErrorCode::SUCCESS);

  if (success) {
    move_group.execute(plan);
  } else {
    RCLCPP_ERROR(node->get_logger(), "Planning failed!");
  }

  // --- Cartesian-space goal (end-effector pose) ---
  geometry_msgs::msg::Pose target_pose;
  target_pose.orientation.w = 1.0;
  target_pose.position.x = 0.4;
  target_pose.position.y = 0.1;
  target_pose.position.z = ____;  // 0.5

  move_group.setPoseTarget(target_pose);
  success = (move_group.plan(plan) ==
             moveit::core::MoveItErrorCode::SUCCESS);
  if (success) move_group.execute(plan);

  spinner.join();
  rclcpp::shutdown();
  return 0;
}

Cartesian Path Planning (Straight-Line Tool Motion)

// cartesian_path.cpp
#include <moveit/move_group_interface/move_group_interface.h>

void plan_cartesian_path(
    moveit::planning_interface::MoveGroupInterface& move_group) {

  // Define waypoints for straight-line motion
  std::vector<geometry_msgs::msg::Pose> waypoints;

  geometry_msgs::msg::Pose start_pose =
      move_group.getCurrentPose().pose;

  // Move 10 cm forward along x
  geometry_msgs::msg::Pose wp1 = start_pose;
  wp1.position.x += 0.1;
  waypoints.push_back(wp1);

  // Move 10 cm upward along z
  geometry_msgs::msg::Pose wp2 = wp1;
  wp2.position.z += 0.1;
  waypoints.push_back(wp2);

  // Move back to start xy, keep z raised
  geometry_msgs::msg::Pose wp3 = wp2;
  wp3.position.x = start_pose.position.x;
  waypoints.push_back(wp3);

  // Compute Cartesian path
  moveit_msgs::msg::RobotTrajectory trajectory;
  const double eef_step = 0.01;       // 1 cm interpolation step
  const double jump_threshold = ____;  // 0.0 (disables jump check)

  double fraction = move_group.computeCartesianPath(
      waypoints, eef_step, jump_threshold, trajectory);

  RCLCPP_INFO(rclcpp::get_logger("cartesian"),
    "Cartesian path: %.2f%% achieved", fraction * 100.0);

  if (fraction > 0.95) {  // Accept if >95% of path planned
    moveit::planning_interface::MoveGroupInterface::Plan cart_plan;
    cart_plan.trajectory_ = trajectory;
    move_group.execute(cart_plan);
  }
}

Add Collision Object to Planning Scene

// add_box.cpp
#include <moveit/planning_scene_interface/planning_scene_interface.h>
#include <moveit_msgs/msg/collision_object.hpp>
#include <shape_msgs/msg/solid_primitive.hpp>

void add_box_obstacle(
    moveit::planning_interface::PlanningSceneInterface& psi) {

  moveit_msgs::msg::CollisionObject box;
  box.header.frame_id = "world";
  box.id = "obstacle_box";

  // Define box shape
  shape_msgs::msg::SolidPrimitive primitive;
  primitive.type = shape_msgs::msg::SolidPrimitive::BOX;
  primitive.dimensions = {0.2, 0.2, 0.4};  // x, y, z in meters

  // Define pose of box
  geometry_msgs::msg::Pose box_pose;
  box_pose.orientation.w = 1.0;
  box_pose.position.x = 0.5;
  box_pose.position.y = 0.0;
  box_pose.position.z = ____;  // 0.2 (half-height off ground)

  box.primitives.push_back(primitive);
  box.primitive_poses.push_back(box_pose);
  box.operation = moveit_msgs::msg::CollisionObject::ADD;

  // Apply to scene — planner will now avoid this box
  psi.applyCollisionObject(box);
}

Grasp & Gripper Integration

// gripper_action.cpp
#include <rclcpp/rclcpp.hpp>
#include <rclcpp_action/rclcpp_action.hpp>
#include <control_msgs/action/gripper_command.hpp>

class GripperController : public rclcpp::Node {
public:
  using GripperCommand = control_msgs::action::GripperCommand;
  using GoalHandle = rclcpp_action::ClientGoalHandle<GripperCommand>;

  GripperController() : Node("gripper_controller") {
    client_ = rclcpp_action::create_client<GripperCommand>(
        this, "/gripper_controller/gripper_cmd");
  }

  void open_gripper() { send_command(0.08, 50.0); }  // 8cm open, 50N
  void close_gripper() { send_command(____,  80.0); }  // 0.0 (fully closed)

private:
  rclcpp_action::Client<GripperCommand>::SharedPtr client_;

  void send_command(double position, double max_effort) {
    if (!client_->wait_for_action_server(std::chrono::seconds(5))) {
      RCLCPP_ERROR(get_logger(), "Gripper action server not available");
      return;
    }
    GripperCommand::Goal goal;
    goal.command.position = position;
    goal.command.max_effort = max_effort;

    auto send_goal_options =
        rclcpp_action::Client<GripperCommand>::SendGoalOptions();
    send_goal_options.result_callback =
        [this](const GoalHandle::WrappedResult& result) {
          if (result.code == rclcpp_action::ResultCode::SUCCEEDED) {
            RCLCPP_INFO(get_logger(), "Gripper command succeeded");
          }
        };
    client_->async_send_goal(goal, send_goal_options);
  }
};

// 1. plan arm to pre_grasp_pose (10 cm above object)
// 2. Cartesian path down 10 cm to grasp_pose
// 3. close_gripper()
// 4. Cartesian path up 10 cm (lift object)
// 5. plan arm to place_pose
// 6. open_gripper()

#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from moveit.planning import MoveItPy
from moveit.core.robot_state import RobotState
import numpy as np

def main():
    rclpy.init()
    node = rclpy.create_node('arm_mover_py')

    # Create MoveItPy instance (loads URDF/SRDF from param server)
    moveit = MoveItPy(node_name="moveit_py")
    arm = moveit.get_planning_component("arm")

    # --- Joint-space goal ---
    arm.set_start_state_to_current_state()
    
    robot_state = RobotState(moveit.get_robot_model())
    robot_state.set_joint_group_positions("arm", [0.0, -1.57, 0.0, -1.57, 0.0, 0.0])
    arm.set_goal_state(robot_state=____)  # robot_state

    plan_result = arm.plan()
    if plan_result:
        moveit.execute(plan_result.trajectory, controllers=[])

    # --- Cartesian-space goal ---
    from geometry_msgs.msg import PoseStamped
    pose_goal = PoseStamped()
    pose_goal.header.frame_id = "world"
    pose_goal.pose.position.x = 0.4
    pose_goal.pose.position.y = 0.1
    pose_goal.pose.position.z = 0.5
    pose_goal.pose.orientation.w = 1.0

    arm.set_goal_state(pose_stamped_msg=pose_goal, pose_link="tool0")
    plan_result = arm.plan()

Benchmark IK Solvers

# ik_benchmark.py — compare KDL vs TRAC-IK success rates
import rclpy, time, random
from rclpy.node import Node
from moveit_msgs.srv import GetPositionIK
from geometry_msgs.msg import PoseStamped

class IKBenchmark(Node):
    def __init__(self):
        super().__init__('ik_benchmark')
        self.client = self.create_client(GetPositionIK, '/compute_ik')
        self.client.wait_for_service()

    def test_ik(self, x, y, z):
        req = GetPositionIK.Request()
        req.ik_request.group_name = 'arm'
        ps = PoseStamped()
        ps.header.frame_id = 'world'
        ps.pose.position.x, ps.pose.position.y, ps.pose.position.z = x, y, z
        ps.pose.orientation.w = 1.0
        req.ik_request.pose_stamped = ps
        req.ik_request.timeout.sec = ____  # 1 second timeout
        future = self.client.call_async(req)
        rclpy.spin_until_future_complete(self, future)
        return future.result().error_code.val == 1  # 1 = SUCCESS

def main():
    rclpy.init()
    bench = IKBenchmark()
    successes = sum(bench.test_ik(
        random.uniform(0.2, 0.6),
        random.uniform(-0.4, 0.4),
        random.uniform(0.1, 0.8)) for _ in range(100))
    bench.get_logger().info(f"IK success rate: {successes}%")
    rclpy.shutdown()

if __name__ == '__main__':
    main()

Common Failures & Fix Patterns

Failure 1: "No IK Solution Found"

Symptom: move_group.plan() fails with IK error; arm cannot reach pose.

Root Cause: KDL diverges near workspace limits or singularities.

Fix: Switch to TRAC-IK and check workspace reachability:

# kinematics.yaml
arm:
  kinematics_solver: trac_ik_kinematics_plugin/TRAC_IKKinematicsPlugin
  kinematics_solver_timeout: 0.005
  solve_type: ____  # Speed
# Visualise reachability workspace
ros2 run moveit_ros_planning_interface \
  moveit_print_planning_model_info
# Check joint limits in URDF — compare to target pose

Failure 2: Cartesian Path Returns Low Fraction (<0.9)

Symptom: computeCartesianPath returns fraction 0.3-0.7; trajectory incomplete.

Root Cause: Path passes through singularity or collision region.

Fix: Reduce step size and break path into segments:

// Finer interpolation catches singularities earlier
double fraction = move_group.computeCartesianPath(
    waypoints,
    0.005,   // Reduce from 0.01 to 0.005 (5 mm steps)
    ____,    // 0.0 (disable jump threshold initially)
    trajectory);

// If still <0.9, split waypoints into two calls and plan with OMPL
// for the gap between them

Failure 3: Gripper Doesn't Close on Object

Symptom: Gripper closes but object falls; grasp not stable.

Root Cause: Grasp pose offset wrong; approach direction incorrect.

Fix: Tune pre-grasp offset and add attach/detach to planning scene:

// After gripper closes, tell MoveIt2 the object is attached
moveit_msgs::msg::AttachedCollisionObject aco;
aco.link_name = "tool0";            // Link holding the object
aco.object.id = "my_object";       // Must match collision object id
aco.object.operation = moveit_msgs::msg::CollisionObject::ADD;
// This prevents MoveIt2 from treating object as obstacle
planning_scene_interface.applyAttachedCollisionObject(aco);

// After placing:
aco.object.operation = moveit_msgs::msg::CollisionObject::____; // REMOVE

Failure 4: Planning Takes Too Long (>5 s)

Symptom: Planner times out; no solution found in time budget.

Root Cause: Path constraints too tight; environment too cluttered; wrong planner.

Fix: Use RRTConnect, loosen tolerances, simplify scene:

move_group.setPlannerId("RRTConnectkConfigDefault");  // Fastest
move_group.setPlanningTime(____);                       // 10.0 seconds (increase budget)
move_group.setGoalPositionTolerance(0.01);             // Loosen from 0.0001 to 0.01 m
move_group.setGoalOrientationTolerance(0.05);          // Loosen orientation
