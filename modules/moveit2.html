MoveIt2

Plan and execute collision-free arm motions using MoveIt2's motion planning framework.

Motion Planning RViz2 C++ Heavy

What / Why / When

What

MoveIt2 is the premier motion planning framework for ROS2. It provides collision-aware trajectory planning for robotic arms and manipulators. Core components: MoveGroupInterface (C++ API), PlanningScene (world model with obstacles), OMPL/CHOMP/PILZ planners (compute collision-free paths), KDL/KDL-IK/BioIK kinematics solvers, and RViz2 Motion Planning plugin. MoveIt2 wraps these components into action servers that accept goal poses and return joint trajectories. The robot URDF + SRDF define the kinematic chain, joint limits, and planning groups. MoveIt2 natively integrates with ROS2 controllers via FollowJointTrajectory action.

Why

Manual arm trajectory planning is intractable: 6-DOF robots have complex joint interactions, obstacle avoidance in 6D configuration space requires sampling-based algorithms (RRT, PRM), and inverse kinematics has multiple solutions requiring selection logic. MoveIt2 solves all of this: OMPL planners handle high-dimensional planning, FCL collision checking prevents crashes, and IK solvers convert Cartesian goals to joint angles. Without MoveIt2, building safe manipulation requires months of custom planning code. With it, you plan to a Cartesian pose in 3 lines of C++.

When

Use MoveIt2 for any robotic arm manipulation: pick-and-place, assembly tasks, welding paths, and surgical robotics. Use it when you need collision-aware planning (workspace with obstacles), Cartesian planning (straight-line end-effector motion), or joint-space planning (move specific joints). Don't use MoveIt2 for mobile base navigation (use Nav2 instead) or for simple joint trajectory following without planning. For time-critical control loops under 10ms, use custom controllers; MoveIt2 planning latency is 50–500ms.

10-Minute Drill

C++ Micro-Task: Move to Named Target

Plan and execute motion to a pre-defined named configuration (e.g., "home"):

#include <rclcpp/rclcpp.hpp>
#include <moveit/move_group_interface/move_group_interface.h>

int main(int argc, char* argv[]) {
  rclcpp::init(argc, argv);
  auto node = std::make_shared<rclcpp::Node>("moveit_drill");
  auto logger = node->get_logger();

  // Spin node in background thread for MoveIt callbacks
  rclcpp::executors::SingleThreadedExecutor executor;
  executor.add_node(node);
  auto spin_thread = std::thread([&executor]() { executor.spin(); });

  // Connect to planning group (name defined in SRDF)
  moveit::planning_interface::MoveGroupInterface move_group(node, "____");  // "arm"

  // Option A: Move to named target
  move_group.setNamedTarget("____");  // "home"
  moveit::planning_interface::MoveGroupInterface::Plan plan;
  bool success = (move_group.plan(plan) == moveit::core::MoveItErrorCode::SUCCESS);
  RCLCPP_INFO(logger, "Plan %s", success ? "SUCCEEDED" : "FAILED");

  if (success) {
    move_group.____();  // execute()
  }

  rclcpp::shutdown();
  spin_thread.join();
  return 0;
}

SRDF — Planning Groups

<!-- my_robot.srdf -->
<robot name="my_robot">

  <!-- Define planning group for the arm -->
  <group name="arm">
    <chain base_link="base_link" tip_link="wrist_link"/>
  </group>

  <!-- Define planning group for gripper -->
  <group name="gripper">
    <joint name="finger_joint_left"/>
    <joint name="finger_joint_right"/>
  </group>

  <!-- Named positions for setNamedTarget() -->
  <group_state name="home" group="arm">
    <joint name="joint1" value="0"/>
    <joint name="joint2" value="-1.5708"/>
    <joint name="joint3" value="0"/>
    <joint name="joint4" value="-1.5708"/>
    <joint name="joint5" value="0"/>
    <joint name="joint6" value="0"/>
  </group_state>

  <!-- Self-collision pairs to ignore (speed up planning) -->
  <disable_collisions link1="link1" link2="base_link" reason="Adjacent"/>

</robot>

Planning Scene: Add Collision Object

#include <moveit/planning_scene_interface/planning_scene_interface.h>
#include <moveit_msgs/msg/collision_object.hpp>
#include <shape_msgs/msg/solid_primitive.hpp>

moveit::planning_interface::PlanningSceneInterface psi;

moveit_msgs::msg::CollisionObject box;
box.header.frame_id = "base_link";
box.id = "obstacle_box";

shape_msgs::msg::SolidPrimitive primitive;
primitive.type = shape_msgs::msg::SolidPrimitive::BOX;
primitive.dimensions = {0.1, 0.1, 0.5};  // x, y, z in meters

geometry_msgs::msg::Pose box_pose;
box_pose.position.x = 0.5;
box_pose.position.y = 0.0;
box_pose.position.z = 0.25;
box_pose.orientation.w = 1.0;

box.primitives.push_back(primitive);
box.primitive_poses.push_back(box_pose);
box.operation = moveit_msgs::msg::CollisionObject::ADD;

psi.applyCollisionObject(box);

Common Failures & Fix Patterns

Failure 1: Planning Failed - No IK Solution

Symptom: Failed to find IK solution in logs, plan returns FAILURE.

Root Cause: Target pose is outside the robot's workspace, or the IK solver has numerical singularities near the target.

Fix: Check workspace limits and try nearby poses:

// Check if pose is reachable before planning
bool found_ik = move_group.setApproximateJointValueTarget(target_pose, "end_effector");
RCLCPP_INFO(logger, "IK %s", found_ik ? "FOUND" : "NOT FOUND");

// Try with tolerance for approximate IK
move_group.setGoalOrientationTolerance(0.1);  // 0.1 rad tolerance
move_group.setGoalPositionTolerance(0.01);    // 1cm tolerance

Failure 2: Motion Planning Timeout

Symptom: Planning takes too long or times out with TIMED_OUT error.

Root Cause: Narrow passages in configuration space, too many collision objects, or planner requires more time.

Fix: Increase planning time or switch planners:

// Increase planning time (default: 5.0 seconds)
move_group.setPlanningTime(15.0);

// Try a different planner
move_group.setPlannerId("RRTstar");  // More optimal but slower
// or
move_group.setPlannerId("LazyPRMkConfigDefault");  // Good for repeated queries

Failure 3: Collision Object Not Detected

Symptom: Robot plans through known obstacles in the scene.

Root Cause: Collision object not added to planning scene, or frame_id mismatch.

Fix: Verify planning scene and check frame IDs:

# Check current planning scene contents
ros2 service call /get_planning_scene moveit_msgs/srv/GetPlanningScene \
  "{components: {components: 1}}"

# Expected output shows all registered collision objects
// Verify frame_id matches robot base frame
box.header.frame_id = move_group.getPlanningFrame();
// getPlanningFrame() returns "base_link" or "world" - match this exactly

Exercise: Force a Planning Failure

Add a collision box blocking the entire workspace, then observe planning failure and recovery:

// Step 1: Add an impenetrable wall in front of the robot
moveit_msgs::msg::CollisionObject wall;
wall.header.frame_id = "base_link";
wall.id = "blocking_wall";

shape_msgs::msg::SolidPrimitive wall_prim;
wall_prim.type = shape_msgs::msg::SolidPrimitive::BOX;
wall_prim.dimensions = {0.02, 2.0, 2.0};  // thin tall wall

geometry_msgs::msg::Pose wall_pose;
wall_pose.position.x = 0.3;
wall_pose.position.y = 0.0;
wall_pose.position.z = 1.0;
wall_pose.orientation.w = 1.0;

wall.primitives.push_back(wall_prim);
wall.primitive_poses.push_back(wall_pose);
wall.operation = moveit_msgs::msg::CollisionObject::ADD;
psi.applyCollisionObject(wall);

// Step 2: Try to plan to pose on the other side of the wall
// Observe: plan() returns PLANNING_FAILED

// Step 3: Remove wall and replan
wall.operation = moveit_msgs::msg::CollisionObject::____; // REMOVE
psi.applyCollisionObject(wall);
// Observe: plan() now succeeds

Detection: Check MoveIt2 logs for collision information:

ros2 run moveit2_drill moveit_drill_node --ros-args --log-level DEBUG
# Look for: "Collision between... and blocking_wall"
Warning:

MoveIt2 planning is non-deterministic with sampling-based planners (RRT, PRM). Running the same plan twice may give different trajectories. Use deterministic planners (PILZ) for repeatable industrial paths.
