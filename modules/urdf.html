URDF Models

Define robot geometry, links, joints, and inertials for simulation and visualization.

Needs RViz Required

What / Why / When

What

URDF (Unified Robot Description Format) is XML defining robot structure. It specifies links (rigid bodies), joints (connections), collision/visual geometry, and physical properties (mass, inertia). robot_state_publisher converts URDF to TF transforms. Xacro extends URDF with macros, properties, and includes for reusable components. URDF drives RViz visualization, Gazebo simulation, and motion planning collision checking.

Why

Manual TF broadcasting for multi-link robots is error-prone. URDF provides single source of truth for robot structure. It enables automatic TF tree generation, collision avoidance in planners, and physically accurate simulation. Proper inertials prevent physics explosions in Gazebo. Visual meshes create realistic rendering. URDF is required for Nav2's costmap footprint and manipulation planning.

When

Create URDF for any robot with multiple rigid bodies. Start with simple boxes for rapid prototyping, add meshes later. Use xacro for robots with repeated structures (wheels, sensors). Include inertials only when using Gazebo physics. Keep separate visual and collision geometries - simplified collision meshes improve performance. Version control URDF alongside code for reproducible testing.

10-Minute Drill

Minimal 2-Link Robot (URDF)

Create a simple robot with base and laser:

<?xml version="1.0"?>
<robot name="simple_robot">
  
  <!-- Base link (robot body) -->
  <link name="____">  <!-- base_link -->
    <visual>
      <geometry>
        <box size="0.5 0.3 0.1"/>  <!-- length width height in meters -->
      </geometry>
      <material name="blue">
        <color rgba="0 0 1 1"/>  <!-- R G B Alpha -->
      </material>
    </visual>
    
    <collision>
      <geometry>
        <box size="0.5 0.3 0.1"/>
      </geometry>
    </collision>
    
    <inertial>
      <mass value="____"/>  <!-- 10.0 kg -->
      <inertia ixx="0.1" ixy="0.0" ixz="0.0"
               iyy="0.1" iyz="0.0"
               izz="0.1"/>
    </inertial>
  </link>
  
  <!-- Laser link -->
  <link name="____">  <!-- laser_frame -->
    <visual>
      <geometry>
        <cylinder radius="0.05" length="0.05"/>
      </geometry>
      <material name="red">
        <color rgba="1 0 0 1"/>
      </material>
    </visual>
    
    <collision>
      <geometry>
        <cylinder radius="0.05" length="0.05"/>
      </geometry>
    </collision>
    
    <inertial>
      <mass value="0.5"/>
      <inertia ixx="0.001" ixy="0.0" ixz="0.0"
               iyy="0.001" iyz="0.0"
               izz="0.001"/>
    </inertial>
  </link>
  
  <!-- Joint connecting base to laser -->
  <joint name="base_to_laser" type="____">  <!-- fixed -->
    <parent link="____"/>  <!-- base_link -->
    <child link="____"/>   <!-- laser_frame -->
    <origin xyz="0.1 0 0.15" rpy="0 0 0"/>  <!-- x y z roll pitch yaw -->
  </joint>
  
</robot>

Xacro with Properties

Convert to xacro for reusability:

<?xml version="1.0"?>
<robot xmlns:xacro="http://www.ros.org/wiki/xacro" name="simple_robot">
  
  <!-- Properties for easy tuning -->
  <xacro:property name="base_length" value="____"/>  <!-- 0.5 -->
  <xacro:property name="base_width" value="0.3"/>
  <xacro:property name="base_height" value="0.1"/>
  <xacro:property name="base_mass" value="10.0"/>
  
  <xacro:property name="laser_x_offset" value="0.1"/>
  <xacro:property name="laser_z_offset" value="0.15"/>
  
  <!-- Macro for inertia calculation (box) -->
  <xacro:macro name="box_inertia" params="m x y z">
    <inertial>
      <mass value="${____}"/>  <!-- m -->
      <inertia ixx="${m*(y*y+z*z)/12.0}" ixy="0.0" ixz="0.0"
               iyy="${m*(x*x+z*z)/12.0}" iyz="0.0"
               izz="${m*(x*x+y*y)/12.0}"/>
    </inertial>
  </xacro:macro>
  
  <link name="base_link">
    <visual>
      <geometry>
        <box size="${base_length} ${base_width} ${base_height}"/>
      </geometry>
      <material name="blue">
        <color rgba="0 0 1 1"/>
      </material>
    </visual>
    <collision>
      <geometry>
        <box size="${base_length} ${base_width} ${base_height}"/>
      </geometry>
    </collision>
    <xacro:box_inertia m="${base_mass}" 
                        x="${base_length}" 
                        y="${base_width}" 
                        z="${base_height}"/>
  </link>
  
  <link name="laser_frame">
    <visual>
      <geometry>
        <cylinder radius="0.05" length="0.05"/>
      </geometry>
      <material name="red">
        <color rgba="1 0 0 1"/>
      </material>
    </visual>
  </link>
  
  <joint name="base_to_laser" type="fixed">
    <parent link="base_link"/>
    <child link="laser_frame"/>
    <origin xyz="${laser_x_offset} 0 ${laser_z_offset}" rpy="0 0 0"/>
  </joint>
  
</robot>

Concept ↔ Code (URDF Structure)

Link Elements

Element Purpose Required
<visual> Appearance in RViz/Gazebo GUI For visualization
<collision> Physics collision detection For simulation/planning
<inertial> Mass and inertia tensor For Gazebo physics

Joint Types

Type Degrees of Freedom Use Case
fixed 0 Sensor mounts, static attachments
revolute 1 (rotation) Hinges, wheels with limits
continuous 1 (rotation, no limits) Freely spinning wheels
prismatic 1 (translation) Linear actuators, elevators
floating 6 Rarely used (base link in some models)
planar 2 Rarely used

Geometry Primitives

<!-- Box -->
<geometry>
  <box size="length width height"/>
</geometry>

<!-- Cylinder -->
<geometry>
  <cylinder radius="0.1" length="0.5"/>
</geometry>

<!-- Sphere -->
<geometry>
  <sphere radius="0.2"/>
</geometry>

<!-- Mesh (STL, DAE, OBJ) -->
<geometry>
  <mesh filename="package://my_robot/meshes/base.stl" scale="1 1 1"/>
</geometry>

Parameters & Config

Robot State Publisher Launch

from launch import LaunchDescription
from launch_ros.actions import Node
import os
from ament_index_python.packages import get_package_share_directory

def generate_launch_description():
    # Load URDF file
    urdf_file = os.path.join(
        get_package_share_directory('my_robot_description'),
        'urdf',
        'robot.urdf.xacro'
    )
    
    # Process xacro to URDF
    from launch.actions import ExecuteProcess
    xacro_process = ExecuteProcess(
        cmd=['xacro', urdf_file],
        output='screen'
    )
    
    # Or read directly if pure URDF
    with open(urdf_file, 'r') as f:
        robot_description = f.read()
    
    return LaunchDescription([
        Node(
            package='robot_state_publisher',
            executable='robot_state_publisher',
            name='robot_state_publisher',
            output='screen',
            parameters=[{
                'robot_description': robot_description,
                'publish_frequency': 30.0
            }]
        ),
        
        # Joint state publisher (for testing without real hardware)
        Node(
            package='joint_state_publisher_gui',
            executable='joint_state_publisher_gui',
            name='joint_state_publisher_gui'
        )
    ])

Inertia Calculations (Common Shapes)

<!-- Box inertia (m=mass, x/y/z=dimensions) -->
<inertia ixx="${m*(y*y+z*z)/12.0}" ixy="0.0" ixz="0.0"
         iyy="${m*(x*x+z*z)/12.0}" iyz="0.0"
         izz="${m*(x*x+y*y)/12.0}"/>

<!-- Cylinder inertia (axis along z, r=radius, h=height) -->
<inertia ixx="${m*(3*r*r+h*h)/12.0}" ixy="0.0" ixz="0.0"
         iyy="${m*(3*r*r+h*h)/12.0}" iyz="0.0"
         izz="${m*r*r/2.0}"/>

<!-- Sphere inertia (r=radius) -->
<inertia ixx="${2*m*r*r/5.0}" ixy="0.0" ixz="0.0"
         iyy="${2*m*r*r/5.0}" iyz="0.0"
         izz="${2*m*r*r/5.0}"/>

Gazebo-Specific Extensions

<robot>
  <!-- Standard URDF link -->
  <link name="base_link">...</link>
  
  <!-- Gazebo material/friction/physics -->
  <gazebo reference="base_link">
    <material>Gazebo/Blue</material>
    <mu1>0.5</mu1>  <!-- Friction coefficient 1 -->
    <mu2>0.5</mu2>  <!-- Friction coefficient 2 -->
    <kp>1000000.0</kp>  <!-- Contact stiffness -->
    <kd>1.0</kd>  <!-- Contact damping -->
  </gazebo>
  
  <!-- Gazebo plugin (sensor, controller) -->
  <gazebo reference="laser_frame">
    <sensor type="ray" name="laser_sensor">
      <update_rate>10</update_rate>
      <ray>
        <scan>
          <horizontal>
            <samples>360</samples>
            <resolution>1.0</resolution>
            <min_angle>-3.14159</min_angle>
            <max_angle>3.14159</max_angle>
          </horizontal>
        </scan>
        <range>
          <min>0.1</min>
          <max>10.0</max>
        </range>
      </ray>
      <plugin name="laser_controller" filename="libgazebo_ros_ray_sensor.so">
        <ros>
          <remapping>~/out:=scan</remapping>
        </ros>
        <output_type>sensor_msgs/LaserScan</output_type>
        <frame_name>laser_frame</frame_name>
      </plugin>
    </sensor>
  </gazebo>
</robot>

Common Failures & Fix Patterns

Failure 1: Robot Falls Through Ground in Gazebo

Symptom: Robot spawns but immediately falls infinitely in simulation.

Root Cause: Missing or incorrect inertial properties, zero mass, or missing collision geometry.

Fix: Add proper inertials to all links:

<link name="base_link">
  <visual>...</visual>
  
  <!-- Must have collision -->
  <collision>
    <geometry>
      <box size="0.5 0.3 0.1"/>
    </geometry>
  </collision>
  
  <!-- Must have non-zero mass and inertia -->
  <inertial>
    <mass value="10.0"/>  <!-- NOT 0.0 -->
    <inertia ixx="0.1" ixy="0.0" ixz="0.0"
             iyy="0.1" iyz="0.0"
             izz="0.1"/>  <!-- NOT all zeros -->
  </inertial>
</link>

Failure 2: Joint Origin Misalignment

Symptom: Child link appears in wrong position relative to parent in RViz.

Root Cause: Joint origin xyz/rpy incorrectly specified or visual/collision origins don't match.

Fix: Verify origin carefully with correct coordinate system:

<joint name="base_to_sensor" type="fixed">
  <parent link="base_link"/>
  <child link="sensor_link"/>
  <!-- xyz: x forward, y left, z up in parent frame -->
  <!-- rpy: roll(x), pitch(y), yaw(z) in radians -->
  <origin xyz="0.2 0 0.1" rpy="0 0 0"/>
</joint>

<!-- If visual geometry center doesn't match link origin, offset it -->
<link name="sensor_link">
  <visual>
    <origin xyz="0 0 0.025" rpy="0 0 0"/>  <!-- Offset visual center -->
    <geometry>
      <cylinder radius="0.05" length="0.05"/>
    </geometry>
  </visual>
</link>

Failure 3: Xacro Not Expanding

Symptom: ${property} appears literally in error messages instead of expanded value.

Root Cause: Loading xacro file directly without processing through xacro command.

Fix: Process xacro before loading:

# Wrong - loads xacro file directly
with open('robot.urdf.xacro', 'r') as f:
    robot_description = f.read()

# Correct - process xacro first
import subprocess
xacro_output = subprocess.check_output(
    ['xacro', 'robot.urdf.xacro']
).decode('utf-8')
robot_description = xacro_output

# Or use xacro Python API
import xacro
robot_description = xacro.process_file('robot.urdf.xacro').toxml()

Failure Injection Drill

Exercise: Debug Invalid Inertia

Create URDF with intentionally bad inertia and observe Gazebo failure:

<link name="bad_link">
  <visual>
    <geometry><box size="1 1 1"/></geometry>
  </visual>
  <collision>
    <geometry><box size="1 1 1"/></geometry>
  </collision>
  
  <!-- Invalid: diagonal inertia elements must be positive -->
  <inertial>
    <mass value="10.0"/>
    <inertia ixx="-0.1" ixy="0.0" ixz="0.0"
             iyy="0.1" iyz="0.0"
             izz="0.1"/>
  </inertial>
</link>

Detection: Spawn in Gazebo:

ros2 launch gazebo_ros gazebo.launch.py
ros2 run gazebo_ros spawn_entity.py -entity my_robot -file robot.urdf
# Expected: Gazebo error about invalid inertia matrix

Recovery: Use positive definite inertia matrix (all diagonal elements positive, satisfy triangle inequalities).

Warning:

Simplified collision geometry significantly improves Gazebo performance. Use primitive shapes (boxes, cylinders) for collision even if visual uses detailed meshes.
