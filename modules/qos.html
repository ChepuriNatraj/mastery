QoS Policies

Control reliability, durability, and latency for data flows.

CPU-Light Required

What / Why / When

What

Quality of Service (QoS) policies control how DDS delivers messages between publishers and subscribers. Key policies include reliability (best-effort vs reliable), durability (volatile vs transient-local), history (keep-last vs keep-all), depth (queue size), deadline, and lifespan. Publishers and subscribers must have compatible QoS for communication to occur.

Why

Different data types need different delivery guarantees. Sensor data tolerates drops for low latency. Commands need reliable delivery. State messages require durability for late joiners. QoS mismatches are the most common silent failure in ROS2. Understanding QoS prevents data loss and enables performance tuning for wireless networks and high-frequency sensors.

When

Use best-effort for high-frequency sensor data where latest value matters most. Use reliable for commands and state updates. Set transient-local durability for configuration topics that late subscribers need. Increase depth for bursty data. Always check QoS compatibility with ros2 topic info --verbose when topics don't receive data.

C++ Micro-Task

Create publishers with different QoS profiles:

#include "rclcpp/rclcpp.hpp"
#include "sensor_msgs/msg/laser_scan.hpp"
#include "std_msgs/msg/string.hpp"

class QoSDemo : public rclcpp::Node {
public:
  QoSDemo() : Node("qos_demo") {
    // Best-effort for high-frequency sensor data
    auto sensor_qos = rclcpp::QoS(rclcpp::KeepLast(10))
      .best_effort()
      .____();  // durability_volatile
    
    sensor_pub_ = this->create_publisher<sensor_msgs::msg::LaserScan>(
      "scan", ____);
    
    // Reliable with transient-local for state messages
    auto state_qos = rclcpp::QoS(rclcpp::KeepLast(1))
      .____()  // reliable
      .transient_local();
    
    state_pub_ = this->create_publisher<std_msgs::msg::String>(
      "robot_state", state_qos);
    
    // System default QoS (compatible with most ROS tools)
    auto default_qos = rclcpp::____();
    
    cmd_pub_ = this->create_publisher<std_msgs::msg::String>(
      "cmd_vel", default_qos);
    
    // Custom QoS with deadline
    auto deadline_qos = rclcpp::QoS(rclcpp::KeepLast(10))
      .reliable()
      .deadline(std::chrono::milliseconds(____));  // Max time between messages
    
    heartbeat_pub_ = this->create_publisher<std_msgs::msg::String>(
      "heartbeat", deadline_qos);
  }

private:
  rclcpp::Publisher<sensor_msgs::msg::LaserScan>::SharedPtr sensor_pub_;
  rclcpp::Publisher<std_msgs::msg::String>::SharedPtr state_pub_;
  rclcpp::Publisher<std_msgs::msg::String>::SharedPtr cmd_pub_;
  rclcpp::Publisher<std_msgs::msg::String>::SharedPtr heartbeat_pub_;
};

int main(int argc, char **argv) {
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<QoSDemo>());
  rclcpp::shutdown();
  return 0;
}

Python Micro-Task

Create matching subscribers with QoS compatibility:

import rclpy
from rclpy.node import Node
from rclpy.qos import QoSProfile, ReliabilityPolicy, DurabilityPolicy, HistoryPolicy
from sensor_msgs.msg import LaserScan
from std_msgs.msg import String

class QoSSubscriber(Node):
    def __init__(self):
        super().__init__('qos_subscriber')
        
        # Best-effort for sensor data
        sensor_qos = QoSProfile(
            depth=____,
            reliability=ReliabilityPolicy.____,  # BEST_EFFORT
            durability=DurabilityPolicy.VOLATILE
        )
        
        self.sensor_sub = self.create_subscription(
            LaserScan, 'scan', self.sensor_callback, ____)
        
        # Reliable transient-local for state
        state_qos = QoSProfile(
            depth=1,
            reliability=ReliabilityPolicy.____,  # RELIABLE
            durability=DurabilityPolicy.____  # TRANSIENT_LOCAL
        )
        
        self.state_sub = self.create_subscription(
            String, 'robot_state', self.state_callback, state_qos)
        
        # System default
        from rclpy.qos import qos_profile_system_default
        self.cmd_sub = self.create_subscription(
            String, 'cmd_vel', self.cmd_callback, ____)
    
    def sensor_callback(self, msg):
        self.get_logger().info('Received scan with %d ranges' % len(msg.ranges))
    
    def state_callback(self, msg):
        self.get_logger().info('State: %s' % msg.data)
    
    def cmd_callback(self, msg):
        self.get_logger().info('Command: %s' % msg.data)

def main(args=None):
    rclpy.init(args=args)
    node = QoSSubscriber()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

Parameters & Config

QoS Policy Overview

Policy Options Use Case
Reliability BEST_EFFORT, RELIABLE RELIABLE for commands, BEST_EFFORT for sensors
Durability VOLATILE, TRANSIENT_LOCAL TRANSIENT_LOCAL for late joiners needing last value
History KEEP_LAST(n), KEEP_ALL KEEP_LAST(10) typical, KEEP_ALL for guaranteed processing
Deadline Duration Detect missing periodic messages
Lifespan Duration Expire old messages in queue
Liveliness AUTOMATIC, MANUAL Detect dead publishers

Programmatic QoS Event Handling

// Set up QoS event callback for incompatible QoS
rclcpp::PublisherOptions pub_options;
pub_options.event_callbacks.incompatible_qos_callback =
  [this](rclcpp::QOSOfferedIncompatibleQoSInfo & info) {
    RCLCPP_WARN(this->get_logger(),
                "Incompatible QoS: %d incompatible connections", 
                info.total_count);
  };

publisher_ = this->create_publisher<MsgType>("topic", qos, pub_options);

Start with SystemDefaultQoS for development. Profile and optimize QoS only when you hit specific issues like drops, latency, or late-joiner problems.

Common Failures & Fix Patterns

Failure 1: Silent Communication Failure

Symptom: Publisher and subscriber both run, but no messages received. No error messages.

Root Cause: QoS incompatibility between publisher (RELIABLE) and subscriber (BEST_EFFORT).

Fix: Match QoS policies or make subscriber more permissive:

# Option 1: Match exactly
pub_qos = QoSProfile(depth=10, reliability=ReliabilityPolicy.RELIABLE)
sub_qos = QoSProfile(depth=10, reliability=ReliabilityPolicy.RELIABLE)

# Option 2: Make subscriber compatible with both
sub_qos = QoSProfile(depth=10, reliability=ReliabilityPolicy.BEST_EFFORT)
# This subscribes to both RELIABLE and BEST_EFFORT publishers

Failure 2: Late Joiner Misses Initial State

Symptom: Subscriber started after publisher misses critical initialization message.

Root Cause: Both use VOLATILE durability, which doesn't store messages.

Fix: Use TRANSIENT_LOCAL durability on both sides:

// Publisher keeps last message for late joiners
auto qos = rclcpp::QoS(rclcpp::KeepLast(1))
  .reliable()
  .transient_local();

auto pub = this->create_publisher<std_msgs::msg::String>("state", qos);

// Subscriber gets last published message immediately upon subscription
auto sub = this->create_subscription<std_msgs::msg::String>(
  "state", qos, callback);

Failure 3: Message Queue Overflow

Symptom: Messages dropped with "queue depth exceeded" warning.

Root Cause: Publisher sends faster than subscriber processes; queue depth too small.

Fix: Increase depth or use KEEP_ALL history:

# Option 1: Larger fixed queue
qos = QoSProfile(depth=100, history=HistoryPolicy.KEEP_LAST)

# Option 2: Keep all (unbounded, use carefully)
qos = QoSProfile(history=HistoryPolicy.KEEP_ALL)

# Option 3: Add lifespan to expire old messages
from rclpy.duration import Duration
qos.lifespan = Duration(seconds=5)  # Discard messages older than 5s

Failure Injection Drill

Exercise: Detect and Recover from QoS Mismatch

Intentionally create incompatible QoS and detect it:

# Publisher with RELIABLE
pub_qos = QoSProfile(depth=10, reliability=ReliabilityPolicy.RELIABLE)
self.publisher = self.create_publisher(String, 'test_topic', pub_qos)

# Subscriber with BEST_EFFORT (incompatible)
sub_qos = QoSProfile(depth=10, reliability=ReliabilityPolicy.BEST_EFFORT)

# Add QoS event callback to detect mismatch
from rclpy.qos import QoSProfile, QoSSubscriptionEventType
sub_options = rclpy.subscription.SubscriptionOptions()
sub_options.event_callbacks = [
    (QoSSubscriptionEventType.REQUESTED_INCOMPATIBLE_QOS,
     lambda event: self.get_logger().error(
         f'Incompatible QoS detected! Policy: {event.last_policy_kind}'))
]

self.subscription = self.create_subscription(
    String, 'test_topic', self.callback, sub_qos,
    subscription_options=sub_options)

Detection: Run both nodes and check logs:

ros2 run my_pkg publisher
ros2 run my_pkg subscriber
# Expected: Error log showing incompatible QoS detected

Recovery: Change subscriber to RELIABLE or publisher to BEST_EFFORT.

Warning:

Bag playback often fails due to QoS mismatches. Use ros2 bag play --qos-profile-overrides-path with YAML to force compatible QoS.
