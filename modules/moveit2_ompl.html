OMPL Planners: RRT vs PRM

Understand exactly what RRT and PRM do in MoveIt2, and pick the right planner based on query pattern, speed, and path quality needs.

What / Why / When

What

OMPL is MoveIt2’s planner backend. It uses sampling-based planning in high-dimensional joint space. RRT family planners grow trees from start (and goal for bidirectional variants). PRM family planners build a reusable roadmap graph and query shortest paths on it.

Why

Manipulators have 6+ DOF and nonlinear constraints. Grid planners explode combinatorially. Sampling-based methods are practical and robust in these spaces.

When

Use RRTConnect for fast one-shot goals, PRM for repeated queries in static scenes, and RRT*/PRM* when path optimality matters more than speed.

Planner Selection Cheatsheet

ScenarioBest PlannerReason
Single pick target nowRRTConnectFastest first solution
Many repeated goals in same cellPRMRoadmap amortizes cost
Need shorter/smoother pathRRT* / PRM*Asymptotically optimal
Tight clutter / narrow passageEST / BKPIECEBetter exploratory bias

10-Minute Drill

# ompl_planning.yaml
planner_configs:
  RRTConnectkConfigDefault:
    type: geometric::RRTConnect
    range: 0.0
  PRMkConfigDefault:
    type: geometric::PRM
    max_nearest_neighbors: 10

arm:
  default_planner_config: ____  # RRTConnectkConfigDefault
  planner_configs:
    - RRTConnectkConfigDefault
    - PRMkConfigDefault

Concept ↔ Code

moveit::planning_interface::MoveGroupInterface move_group(node, "arm");
move_group.setPlannerId("RRTConnectkConfigDefault");
move_group.setPlanningTime(2.0);

moveit::planning_interface::MoveGroupInterface::Plan plan;
auto ok = move_group.plan(plan) == moveit::core::MoveItErrorCode::SUCCESS;
if (ok) {
  move_group.execute(plan);
}

// Switch planner for repeated-cell benchmarking
move_group.setPlannerId(____); // "PRMkConfigDefault"

Verify

ros2 param get /move_group planning_pipelines
ros2 topic echo /move_group/display_planned_path --once
ros2 run moveit_ros_benchmarks moveit_run_benchmark
