OOP in C++ for Robotics

Use class design, inheritance, and polymorphism to build reusable controllers, sensors, and ROS2 plugin architectures.

What / Why / When

What

Object-oriented design in robotics models each subsystem as a class: drivers, estimators, planners, controllers. Common interfaces are declared as abstract base classes.

Why

You need swappable algorithms and maintainable code. Nav2 and MoveIt2 are plugin-heavy frameworks built around polymorphism and interfaces.

When

Use inheritance for framework/plugin contracts. Prefer composition for internal implementation details to keep modules decoupled and testable.

10-Minute Drill

class SensorBase {
public:
  virtual double read() = 0;
  virtual std::string name() const = 0;
  virtual ~SensorBase() = default;
};

class LidarSensor : public SensorBase {
public:
  double read() override { return 4.2; }
  std::string name() const override { return "LidarSensor"; }
};

class ImuSensor : public SensorBase {
public:
  double read() override { return 9.81; }
  std::string name() const override { return ____; } // "ImuSensor"
};

Concept ↔ Code (Plugin Style)

class ControllerBase {
public:
  virtual void configure(const rclcpp::Node::SharedPtr& node) = 0;
  virtual geometry_msgs::msg::Twist computeCommand() = 0;
  virtual ~ControllerBase() = default;
};

class FollowPathController : public ControllerBase {
public:
  void configure(const rclcpp::Node::SharedPtr& node) override { node_ = node; }
  geometry_msgs::msg::Twist computeCommand() override {
    geometry_msgs::msg::Twist cmd;
    cmd.linear.x = 0.2;
    return cmd;
  }
private:
  rclcpp::Node::SharedPtr node_;
};

std::unique_ptr<ControllerBase> controller = std::make_unique<FollowPathController>();
controller->configure(node);

Common Failures & Fix Patterns

  • Missing virtual destructor: causes undefined behavior on delete through base pointer.
  • Inheritance overuse: too deep hierarchies. Prefer composition.
  • Interface drift: add override to catch signature mistakes.
  • Tight coupling: depend on interfaces, not concrete classes.
