Bag Recording

Record, replay, and analyze robot data for debugging and benchmarking.

Data Logging Debugging

What / Why / When

What

ROS2 bags record and replay topic messages. The bag format stores timestamped messages in a SQLite3 or MCAP database. Bags preserve message type info, QoS profiles, and topic metadata. They support selective recording (specific topics), compression (zstd/lz4), and splitting (max size/duration). Bags enable offline analysis, regression testing, and data sharing without re-running robot hardware.

Why

Robot testing is non-deterministic - sensor noise, timing variations, and environmental changes make bugs hard to reproduce. Bags capture exact message sequences for deterministic replay. They enable post-processing (SLAM, perception pipelines), algorithm benchmarking on consistent data, and collaborative debugging. Bags reduce hardware dependency during development and support CI/CD testing with real-world data.

When

Record bags during all robot tests, especially when bugs occur. Use bags for algorithm validation, performance profiling, and training ML models. Replay bags to test code changes without hardware. Be aware of disk space - sensor data (images, lidar) fills quickly. Use compression and selective recording. Bags don't preserve service calls or parameter changes by default. QoS incompatibility during playback can cause silent failures.

10-Minute Drill

Record a Bag

Capture robot data from multiple topics:

# Record specific topics
ros2 bag record /scan /odom /cmd_vel /camera/image_raw

# Record all topics (careful - generates huge files)
ros2 bag record -a

# Record with output name and compression
ros2 bag record -o my_test_run \
    --compression-mode file \  # or 'message'
    --compression-format zstd \  # or 'lz4'
    /scan /odom /tf /tf_static

# Record with max duration or size
ros2 bag record -o session \
    --max-bag-duration ____ \  # 300 (5 minutes in seconds)
    --max-bag-size ____ \  # 1000000000 (1GB in bytes)
    -a

# Record with regex pattern
ros2 bag record -e "/camera/.+" "/lidar/.+"
# Records all topics under /camera/* and /lidar/*

Inspect Bag Contents

Analyze recorded data:

# Show bag metadata
ros2 bag info my_test_run

# Output example:
# Files:             my_test_run_0.db3
# Duration:          120.5s
# Start:             Jan 10 2024 14:30:00
# End:               Jan 10 2024 14:32:00
# Messages:          12450
# Topic information:
#   /scan: 1200 msgs (sensor_msgs/msg/LaserScan)
#   /odom: 6000 msgs (nav_msgs/msg/Odometry)
#   /cmd_vel: 500 msgs (geometry_msgs/msg/Twist)
#   /tf: 4750 msgs (tf2_msgs/msg/TFMessage)

Replay a Bag

Play back recorded data:

# Basic replay
ros2 bag play my_test_run

# Play at different speed
ros2 bag play my_test_run --rate ____  # 0.5 (half speed)

# Loop playback
ros2 bag play my_test_run --loop

# Play specific topics only
ros2 bag play my_test_run \
    --topics /scan /odom

# Remap topics during playback
ros2 bag play my_test_run \
    --remap ____:/scan_sim  # /scan:/scan_sim
    --remap /odom:____  # /odom_sim

# Start from specific timestamp
ros2 bag play my_test_run \
    --start-offset 30.0  # Skip first 30 seconds

# Pause at start (press space to continue)
ros2 bag play my_test_run --pause

Concept ↔ Code (Programmatic API)

Recording from Python

from rclpy.node import Node
from rosbag2_py import Writer, StorageOptions, ConverterOptions, TopicMetadata
from sensor_msgs.msg import LaserScan
import rclpy

class BagRecorderNode(Node):
    def __init__(self):
        super().__init__('bag_recorder')
        
        # Setup bag writer
        storage_options = StorageOptions(
            uri='my_bag',
            storage_id='sqlite3'  # or 'mcap'
        )
        converter_options = ConverterOptions(
            input_serialization_format='cdr',
            output_serialization_format='____'  # cdr
        )
        
        self.writer = Writer()
        self.writer.open(storage_options, converter_options)
        
        # Add topic metadata
        topic_info = TopicMetadata(
            name='/scan',
            type='sensor_msgs/msg/LaserScan',
            serialization_format='cdr'
        )
        self.writer.create_topic(topic_info)
        
        # Subscribe to topic
        self.subscription = self.create_subscription(
            LaserScan,
            '/scan',
            self.scan_callback,
            ____  # 10
        )
        
    def scan_callback(self, msg):
        # Write message to bag
        timestamp = self.get_clock().now().nanoseconds
        self.writer.write(
            '/scan',
            ____,  # serialize(msg) - use rclpy.serialization.serialize_message
            timestamp
        )
        
    def __del__(self):
        self.writer.close()

def main():
    rclpy.init()
    node = BagRecorderNode()
    rclpy.spin(node)
    rclpy.shutdown()

Reading Bag in C++

#include <rosbag2_cpp/reader.hpp>
#include <rosbag2_cpp/storage_options.hpp>
#include <rosbag2_cpp/converter_options.hpp>
#include <rclcpp/serialization.hpp>
#include <sensor_msgs/msg/laser_scan.hpp>

int main() {
    // Setup reader
    rosbag2_cpp::Reader reader;
    rosbag2_storage::StorageOptions storage_options;
    storage_options.uri = "my_bag";
    storage_options.storage_id = "sqlite3";
    
    rosbag2_cpp::ConverterOptions converter_options;
    converter_options.input_serialization_format = "cdr";
    converter_options.output_serialization_format = "cdr";
    
    reader.open(storage_options, converter_options);
    
    // Read messages
    while (reader.has_next()) {
        auto serialized_msg = reader.read_next();
        
        if (serialized_msg->topic_name == "/scan") {
            // Deserialize
            rclcpp::SerializedMessage serialized(
                *serialized_msg->serialized_data
            );
            
            sensor_msgs::msg::LaserScan scan;
            rclcpp::Serialization<sensor_msgs::msg::LaserScan> serializer;
            serializer.deserialize_message(&serialized, &scan);
            
            // Process scan message
            std::cout << "Scan with " << scan.ranges.size() 
                      << " points at time " 
                      << serialized_msg->time_stamp << std::endl;
        }
    }
    
    return 0;
}

Launch File Integration

from launch import LaunchDescription
from launch.actions import ExecuteProcess
from launch_ros.actions import Node

def generate_launch_description():
    # Start bag recording in launch
    bag_record = ExecuteProcess(
        cmd=[
            'ros2', 'bag', 'record',
            '-o', 'test_session',
            '--compression-mode', 'file',
            '--compression-format', 'zstd',
            '/scan', '/odom', '/tf', '/tf_static'
        ],
        output='screen'
    )
    
    # Your nodes
    my_node = Node(
        package='my_pkg',
        executable='my_node',
        name='my_node'
    )
    
    return LaunchDescription([
        bag_record,
        my_node
    ])

Parameters & Config

Storage Formats

Format Pros Cons
sqlite3 Default, wide support, good tooling Slower than MCAP, larger file size
mcap Faster write, better compression, industry standard Less mature tooling in ROS2

Compression Formats

Format Speed Ratio Use Case
zstd Fast High General purpose, best default
lz4 Very fast Medium High-bandwidth recording (cameras)
None Instant 1:1 Maximum recording rate, post-process compression

QoS Profile Matching

# When playing back, bag player publishes with RECORDED QoS by default
# This can cause compatibility issues

# Override playback QoS to match subscribers
ros2 bag play my_bag \
    --qos-profile-overrides-path qos_overrides.yaml

# qos_overrides.yaml
/scan:
  reliability: reliable  # Was best_effort in bag
  durability: transient_local

Data Integrity Checks

# Python script to validate bag
import rosbag2_py
from rclpy.serialization import deserialize_message
from sensor_msgs.msg import LaserScan

reader = rosbag2_py.SequentialReader()
storage_options = rosbag2_py.StorageOptions(uri='test_bag', storage_id='sqlite3')
converter_options = rosbag2_py.ConverterOptions('', '')
reader.open(storage_options, converter_options)

scan_count = 0
while reader.has_next():
    (topic, data, t) = reader.read_next()
    if topic == '/scan':
        msg = deserialize_message(data, LaserScan)
        scan_count += 1
        
        # Validate data
        assert len(msg.ranges) > 0, "Empty scan"
        assert msg.angle_min < msg.angle_max, "Invalid angle range"

print(f"Validated {scan_count} scan messages")
Pro Tip:

Always record /tf and /tf_static when recording sensor data. Many visualization and processing tools require transforms to function correctly.

Common Failures & Fix Patterns

Failure 1: QoS Mismatch During Playback

Symptom: Topics published during playback but subscribers don't receive messages.

Root Cause: Bag recorded with different QoS than subscriber expects (e.g., best-effort vs reliable).

Fix: Check recorded QoS and override if needed:

# Check recorded QoS
ros2 bag info my_bag
# Look at "Offered QoS profiles" for each topic

# If subscriber expects reliable but bag has best_effort:
# Option 1: Override during playback
ros2 bag play my_bag --qos-profile-overrides-path qos_override.yaml

# qos_override.yaml
/scan:
  reliability: reliable

# Option 2: Fix subscriber QoS to match recorded data
# In subscriber code:
rclcpp::QoS qos(10);
qos.best_effort();  # Match recorded profile
auto sub = node->create_subscription<LaserScan>("/scan", qos, callback);

Failure 2: Namespace Collisions

Symptom: Playing back bag while robot is running causes duplicate topics or wrong data routing.

Root Cause: Bag topics have same names as live robot topics.

Fix: Remap topics during playback:

# Play bag data under /sim namespace
ros2 bag play my_bag \
    --remap /scan:=/sim/scan \
    --remap /odom:=/sim/odom \
    --remap /cmd_vel:=/sim/cmd_vel

# Now live robot publishes to /scan, bag publishes to /sim/scan

Failure 3: Disk Space Exhaustion

Symptom: Bag recording stops prematurely with I/O error.

Root Cause: Camera or lidar topics fill disk quickly.

Fix: Use compression and size limits:

# Monitor disk during recording
df -h /path/to/bag/directory

# Set maximum bag size (splits into multiple files)
ros2 bag record -o my_bag \
    --max-bag-size 500000000 \  # 500MB per file
    --compression-mode file \
    --compression-format zstd \
    /scan /odom  # Don't record /camera if not needed

# For analysis, record images at lower rate
ros2 topic hz /camera/image_raw
# If 30 Hz, throttle to 5 Hz:
ros2 run topic_tools throttle messages /camera/image_raw 5.0 /camera/image_throttled

# Then record throttled topic
ros2 bag record /camera/image_throttled /scan /odom

Failure Injection Drill

Exercise: Detect Time Desynchronization

Play bag without simulation time and observe timestamp issues:

# Terminal 1: Play bag WITHOUT clock publishing
ros2 bag play my_bag --clock 0
# Bag doesn't publish to /clock topic

# Terminal 2: Start node with sim time enabled
ros2 run my_pkg my_node --ros-args -p use_sim_time:=true
# Node waits for /clock but never receives it

# Terminal 3: Try to echo topics
ros2 topic echo /scan --once
# Messages published but timestamps are in the past

# Check TF
ros2 run tf2_ros tf2_echo map base_link
# Expected: Lookup fails or extrapolation errors

Detection: Check clock source:

ros2 topic list | grep clock
# If empty, no simulation time being published

ros2 param get /my_node use_sim_time
# If true but no /clock, node can't progress time

Recovery: Enable clock publishing:

ros2 bag play my_bag --clock  # Publishes to /clock
# OR disable sim time in nodes:
ros2 run my_pkg my_node --ros-args -p use_sim_time:=false
Warning:

Bags don't record service calls or action goals. Use --all option to include service introspection data, but this significantly increases bag size.
