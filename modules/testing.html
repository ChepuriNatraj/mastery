Testing (Unit, Integration, Launch)

Build robust test suites for ROS2 nodes, services, and launch systems.

Testing CI/CD

What / Why / When

What

ROS2 testing includes unit tests (individual functions), integration tests (node interactions), and launch tests (full system). C++ uses gtest framework, Python uses pytest. launch_testing spins up nodes, sends messages, and verifies behavior. Test fixtures set up nodes, publishers, subscribers for isolated testing. Mocking tools (rclcpp_lifecycle mocks, test_msgs) simulate dependencies. Coverage tools (gcov, pytest-cov) track tested code. Colcon runs tests with colcon test, results in log/. CI/CD integrates with GitHub Actions, Jenkins using colcon test results.

Why

Untested code breaks during refactoring - tests catch regressions. Integration tests verify pub/sub QoS compatibility, service timeouts, action preemption. Launch tests ensure system-level behavior (navigation, localization) works end-to-end. Tests document expected behavior better than comments. Automated testing in CI prevents bad commits from merging. Coverage reports identify untested code paths. Tests enable confident upgrades (ROS2 Humble → Iron) by revealing breaking changes.

When

Write unit tests for all non-trivial functions (> 10 lines). Integration tests for every pub/sub pair, service, action. Launch tests for critical behaviors (autonomous navigation, emergency stop). Run tests before every commit (colcon test). Add coverage checks to CI (require > 80%). Use TDD (Test-Driven Development) for complex algorithms - write test first, then implementation. Mock hardware interfaces to enable testing without robot. Test failure modes (network drop, sensor timeout, invalid input).

Test Types Comparison

Type Scope Tools Speed Use Case
Unit Test Single function/class gtest, pytest Fast (ms) Math functions, data transforms
Integration Test Node interaction gtest + rclcpp, pytest + rclpy Medium (100ms-1s) Pub/sub, service calls, parameter validation
Launch Test Full system launch_testing Slow (seconds) Navigation, multi-node coordination

C++ Integration Test (Testing Publisher/Subscriber)

// test/test_talker_listener.cpp
#include <gtest/gtest.h>
#include "rclcpp/rclcpp.hpp"
#include "std_msgs/msg/string.hpp"
#include <chrono>

using namespace std::chrono_literals;

class TalkerListenerTest : public ::testing::Test {
protected:
    void SetUp() override {
        rclcpp::init(0, nullptr);
        node_ = rclcpp::Node::make_shared("test_node");
        
        // Create subscriber to listen
        received_ = false;
        sub_ = node_->create_subscription<std_msgs::msg::String>(
            "chatter", 10,
            [this](std_msgs::msg::String::SharedPtr msg) {
                received_data_ = msg->data;
                received_ = true;
            }
        );
        
        // Create publisher to test
        pub_ = node_->create_publisher<std_msgs::msg::String>(
            "chatter", 10
        );
    }
    
    void TearDown() override {
        rclcpp::shutdown();
    }
    
    rclcpp::Node::SharedPtr node_;
    rclcpp::Publisher<std_msgs::msg::String>::SharedPtr pub_;
    rclcpp::Subscription<std_msgs::msg::String>::SharedPtr sub_;
    bool received_;
    std::string received_data_;
};

TEST_F(TalkerListenerTest, MessageReceived) {
    // Publish message
    auto msg = std_msgs::msg::String();
    msg.data = "Hello Test";
    pub_->publish(msg);
    
    // Spin until received or timeout
    auto start = std::chrono::steady_clock::now();
    while (!received_) {
        rclcpp::spin_some(node_);
        
        auto elapsed = std::chrono::steady_clock::now() - start;
        if (elapsed > ____s) {  // 1s timeout
            break;
        }
        
        std::this_thread::sleep_for(10ms);
    }
    
    // Verify
    EXPECT_TRUE(received_);
    EXPECT_EQ(received_data_, "____");  // Hello Test
}

int main(int argc, char** argv) {

          rosdep update

        run: colcon build
      
      - name: Test
        run: colcon test --event-handlers console_direct+
      
      - name: Test Results

      - name: Upload coverage

Common Failures & Fix Patterns

Failure 1: Test Timeout

Symptom: Test hangs, eventually times out.

Root Cause: Spinning forever waiting for message/service that never arrives.

Fix: Add timeout to spin:

// Bad: infinite wait
while (!received_) {
    rclcpp::spin_some(node_);
}

// Good: timeout after 1 second
auto start = std::chrono::steady_clock::now();
while (!received_) {
    rclcpp::spin_some(node_);
    
    auto elapsed = std::chrono::steady_clock::now() - start;
    if (elapsed > ____s) {  // 1
        FAIL() << "Timeout waiting for message";
    }
}

Failure 2: QoS Mismatch in Test

Symptom: Publisher sends, subscriber never receives in test.

Root Cause: QoS profile mismatch (publisher BEST_EFFORT, subscriber RELIABLE).

Fix: Match QoS profiles:

// Publisher in node
auto qos = rclcpp::QoS(10).best_effort();
pub_ = create_publisher<std_msgs::msg::String>("topic", qos);

// Subscriber in test must match
auto qos_test = rclcpp::QoS(10).____ ();  // best_effort
sub_ = node_->create_subscription<std_msgs::msg::String>(
    "topic", qos_test, callback
);

Failure 3: Race Condition

Symptom: Test passes sometimes, fails randomly.

Root Cause: Callback not executed before assertion.

Fix: Wait for condition:

# Bad: check immediately
node.send_request()
assert node.response_received  # Might be False!

# Good: wait for condition
import time
timeout = 2.0
start = time.time()
while not node.response_received:
    rclpy.spin_once(node, timeout_sec=0.1)
    if time.time() - start > timeout:
        assert False, "Timeout waiting for response"

assert node.response_received

Failure Injection Drill

Exercise: Test Without Initialization

Forget to initialize rclpy/rclcpp in test:

# test/test_broken.py - intentional error
import rclpy
from rclpy.node import Node

def test_node_creation():
    # rclpy.init()  # MISSING!
    
    node = Node('test_node')  # Will crash
    assert node is not None
    
    node.destroy_node()
    # rclpy.shutdown()

Observe:

colcon test --packages-select my_pkg
# Error: rclpy.exceptions.NotInitializedException: 
#        context is not initialized

Detection: Test crashes immediately, check logs:

colcon test-result --verbose
# Shows: NotInitializedException

Recovery: Add initialization:

def test_node_creation():
    ____ .init()  # rclpy
    node = Node('test_node')
    assert node is not None
    node.destroy_node()
    rclpy.shutdown()
Warning:

Always pair rclpy.init() with rclpy.shutdown() in tests. Use fixtures (@pytest.fixture) or test classes (setup_class/teardown_class) to avoid forgetting.
