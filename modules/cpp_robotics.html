C++ for Robotics

Master the C++ patterns that power every ROS2 node: OOP, smart pointers, rclcpp, and custom publishers/subscribers.

C++17 rclcpp OOP

What / Why / When

What

ROS2 nodes are predominantly written in C++ using the rclcpp (ROS2 C++ client library). rclcpp wraps DDS communication in a type-safe, modern C++ API. Core constructs: rclcpp::Node (base class for all nodes), Publisher, Subscription, Service, Client, Timer, and CallbackGroup. Nearly every ROS2 internal — Nav2, MoveIt2, ros2_control — is a C++ class inheriting from rclcpp::Node or rclcpp_lifecycle::LifecycleNode. Smart pointers (shared_ptr, unique_ptr) are used throughout instead of raw pointers.

Why

C++ delivers deterministic, low-latency execution critical for real-time control loops. Python adds GIL overhead and unpredictable garbage-collection pauses. Hardware drivers, motion controllers, and sensor processing pipelines need microsecond-level predictability. Understanding C++ OOP is also required to extend Nav2/MoveIt2 with custom plugins — every plugin is a C++ class with a defined interface. Smart pointers prevent memory leaks and dangling-pointer bugs that are especially dangerous in long-running robots.

When

Write ROS2 nodes in C++ for: hardware drivers, real-time controllers, Nav2/MoveIt2 plugins, performance-critical algorithms (SLAM, perception). Use Python for: rapid prototyping, scripting, task-level logic, testing. Master the patterns in this module before writing any MoveIt2 C++ code. Understand shared_ptr before touching any Nav2 plugin — they are the primary ownership mechanism in those stacks.

OOP in C++ — Classes, Inheritance, Polymorphism

// sensor_base.hpp — Abstract base class (interface)
#pragma once
#include <string>

class SensorBase {
public:
  // Pure virtual method — subclasses MUST implement
  virtual double read() = 0;

  // Virtual method with default implementation
  virtual std::string name() const { return "UnnamedSensor"; }

  // Virtual destructor — ALWAYS add when class has virtual methods
  virtual ~SensorBase() = default;
};

// laser_sensor.hpp — Concrete derived class
#include "sensor_base.hpp"

class LaserSensor : public SensorBase {
public:
  explicit LaserSensor(double range_max)
    : range_max_(range_max) {}

  // Override pure virtual — provides implementation
  double read() override {
    // Simulate reading
    return range_max_ * 0.75;
  }

  std::string name() const override { return "LaserSensor"; }

private:
  double range_max_;
};

// imu_sensor.hpp
#include "sensor_base.hpp"

class ImuSensor : public SensorBase {
public:
  double read() override { return 9.81; }  // Returns gravity magnitude
  std::string name() const override { return "____"; }  // "ImuSensor"
};

// Usage — polymorphism through base pointer
#include <vector>
#include <memory>
#include <iostream>

int main() {
  // Store different sensor types in same container
  std::vector<std::shared_ptr<SensorBase>> sensors;
  sensors.push_back(std::make_shared<LaserSensor>(10.0));
  sensors.push_back(std::make_shared<ImuSensor>());

  for (auto& sensor : sensors) {
    std::cout << sensor->name()
              << " reads: " << sensor->read() << "\n";
  }
  // Output:
  // LaserSensor reads: 7.5
  // ImuSensor reads: 9.81
  return 0;
}

Pointers & Memory Management

// memory_patterns.cpp

// --- Raw pointer (avoid in modern C++) ---
int* raw = new int(42);
// MUST manually delete — easy to forget or double-delete
delete raw;

// --- unique_ptr — sole ownership, no copying ---
#include <memory>

auto up = std::make_unique<int>(42);
// Memory freed automatically when up goes out of scope
// Cannot copy unique_ptr:
// auto up2 = up;  // ERROR: deleted copy constructor
auto up2 = std::move(up);  // Transfer ownership — up is now null

// --- shared_ptr — shared ownership, reference counted ---
auto sp1 = std::make_shared<int>(42);
auto sp2 = sp1;  // Both own the int — ref count = 2
{
  auto sp3 = sp1;  // ref count = 3
}  // sp3 destroyed — ref count = 2
// int freed when LAST shared_ptr is destroyed (ref count → 0)

std::cout << "use_count: " << sp1.use_count() << "\n";  // ____  // 2

// --- weak_ptr — observe without owning (breaks circular refs) ---
std::weak_ptr<int> wp = sp1;
if (auto locked = wp.lock()) {  // Check object still alive
  std::cout << "Value: " << *locked << "\n";
}

// --- Stack allocation (fastest, no heap) ---
int stack_val = 42;  // Freed when function returns — prefer this

// Stack vs Heap decision:
// Stack:  small, fixed-size, short-lived  →  local variables
// Heap:   large, dynamic size, long-lived →  shared_ptr / unique_ptr

Smart Pointer Quick Reference

Type Ownership Copy? Move? ROS2 Usage
unique_ptr<T> Sole owner No Yes Plugin instances, hardware drivers
shared_ptr<T> Shared (ref-counted) Yes Yes Nodes, message data, callback groups
weak_ptr<T> Non-owning observer Yes Yes Lifecycle node parent refs in plugins
raw T* Unspecified Yes Yes Avoid — only in C-style APIs

Publisher Node (C++)

// talker.cpp
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>
#include <chrono>

using namespace std::chrono_literals;

class Talker : public rclcpp::Node {
public:
  Talker() : Node("talker"), count_(0) {
    // Create publisher: topic "/chatter", queue depth 10
    publisher_ = create_publisher<std_msgs::msg::String>(
        "____", 10);  // "/chatter"

    // Create timer: fires every 500 ms
    timer_ = create_wall_timer(
        500ms,
        std::bind(&Talker::timer_callback, this));
  }

private:
  void timer_callback() {
    auto msg = std_msgs::msg::String();
    msg.data = "Hello ROS2: " + std::to_string(count_++);
    RCLCPP_INFO(get_logger(), "Publishing: '%s'", msg.data.c_str());
    publisher_->publish(msg);
  }

  // Member variables — stored in the class (outlive callbacks)
  rclcpp::Publisher<std_msgs::msg::String>::SharedPtr publisher_;
  rclcpp::TimerBase::SharedPtr timer_;
  size_t count_;
};

int main(int argc, char** argv) {
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<Talker>());
  rclcpp::shutdown();
  return 0;
}

Publisher + Subscriber in One Node

// relay.cpp — subscribes to /input, republishes to /output
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/float64.hpp>

class Relay : public rclcpp::Node {
public:
  Relay() : Node("relay") {
    sub_ = create_subscription<std_msgs::msg::Float64>(
        "/input", 10,
        [this](const std_msgs::msg::Float64::SharedPtr msg) {
          // Lambda callback (alternative to std::bind)
          auto out = std_msgs::msg::Float64();
          out.data = msg->data * 2.0;  // Double the value
          pub_->publish(out);
          RCLCPP_DEBUG(get_logger(),
            "Relayed %.3f → %.3f", msg->data, out.data);
        });

    pub_ = create_publisher<std_msgs::msg::Float64>(
        "/output", ____);  // 10 (queue depth)
  }

private:
  rclcpp::Subscription<std_msgs::msg::Float64>::SharedPtr sub_;
  rclcpp::Publisher<std_msgs::msg::Float64>::SharedPtr pub_;
};

int main(int argc, char** argv) {
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<Relay>());
  rclcpp::shutdown();
  return 0;
}

Declaring & Reading Parameters in C++

// param_node.cpp
#include <rclcpp/rclcpp.hpp>

class ParamNode : public rclcpp::Node {
public:
  ParamNode() : Node("param_node") {
    // Declare parameters with defaults
    declare_parameter("max_speed", 1.0);
    declare_parameter("robot_name", std::string("turtlebot"));
    declare_parameter("enable_debug", false);

    // Read parameter values
    double max_speed = get_parameter("max_speed").as_double();
    std::string name = get_parameter("____").as_string();  // robot_name
    bool debug = get_parameter("enable_debug").as_bool();

    RCLCPP_INFO(get_logger(),
      "max_speed=%.1f, robot=%s, debug=%s",
      max_speed, name.c_str(), debug ? "true" : "false");

    // Register callback for parameter changes at runtime
    param_callback_handle_ = add_on_set_parameters_callback(
        [this](const std::vector<rclcpp::Parameter>& params) {
          rcl_interfaces::msg::SetParametersResult result;
          result.successful = true;
          for (const auto& p : params) {
            RCLCPP_INFO(get_logger(),
              "Parameter changed: %s", p.get_name().c_str());
          }
          return result;
        });
  }

private:
  OnSetParametersCallbackHandle::SharedPtr param_callback_handle_;
};

Executor Types — Single vs Multi-Threaded

// single_threaded.cpp — all callbacks on one thread
rclcpp::spin(std::make_shared<MyNode>());

// multi_threaded.cpp — parallel callback execution
#include <rclcpp/rclcpp.hpp>

int main(int argc, char** argv) {
  rclcpp::init(argc, argv);
  auto node = std::make_shared<MyNode>();

  // MultiThreadedExecutor: runs callbacks in parallel
  rclcpp::executors::MultiThreadedExecutor executor;
  executor.add_node(node);
  executor.spin();

  rclcpp::shutdown();
  return 0;
}

rclcpp Logger Levels

// Five levels — only messages >= active level are printed
RCLCPP_DEBUG(get_logger(), "Detailed trace: x=%.3f", x);  // Lowest
RCLCPP_INFO(get_logger(), "Normal operation message");
RCLCPP_WARN(get_logger(), "Unexpected but handled condition");
RCLCPP_ERROR(get_logger(), "Error — operation failed: %s", err.c_str());
RCLCPP_FATAL(get_logger(), "Unrecoverable error — shutting down");  // Highest

// One-shot logging (avoids spam in tight loops)
RCLCPP_INFO_ONCE(get_logger(), "This prints only the first time");
RCLCPP_WARN_THROTTLE(get_logger(), *get_clock(), 1000,
    "____");  // Prints max once per 1000 ms

Common Failures & Fix Patterns

Failure 1: Segfault — Using Deleted shared_ptr

Symptom: Node crashes with segfault; backtrace shows use-after-free.

Root Cause: Captured this raw pointer in lambda outlives the node.

Fix: Capture shared_from_this() inside callbacks instead of raw this:

// BAD — raw 'this' might dangle if node destroyed before timer fires
timer_ = create_wall_timer(1s,
    [this]() { /* use this */ });

// GOOD — shared ownership; timer callback keeps node alive
timer_ = create_wall_timer(1s,
    [node = ____()]() {  // shared_from_this()
      node->do_something();
    });

Failure 2: "Cannot publish on a non-active publisher"

Symptom: Publisher throws exception; no messages appear.

Root Cause: Publisher created before rclcpp::init() or after rclcpp::shutdown().

Fix: Always create publishers inside the Node constructor (after rclcpp::init()), never globally:

// BAD — global publisher (created before rclcpp::init)
auto pub = rclcpp::create_publisher<std_msgs::msg::String>(...);  // CRASH

// GOOD — publisher created in Node constructor body
class MyNode : public rclcpp::Node {
public:
  MyNode() : Node("my_node") {
    pub_ = create_publisher<std_msgs::msg::String>("/topic", ____);  // 10
  }
private:
  rclcpp::Publisher<std_msgs::msg::String>::SharedPtr pub_;
};

Failure 3: Callbacks Never Fire

Symptom: Subscriber created, messages published, but callback never called.

Root Cause: Forgot to spin — without a spin call, no callbacks execute.

Fix: Always spin the executor after creating the node:

// Missing spin — subscriber never receives anything
auto node = std::make_shared<Listener>();
// Bug: no rclcpp::spin() call → callbacks never invoked

// Fixed:
auto node = std::make_shared<Listener>();
rclcpp::spin(____);  // node — blocks until shutdown

Failure 4: Memory Leak — Circular shared_ptr

Symptom: Memory grows over time; valgrind shows unreachable blocks.

Root Cause: Two objects hold shared_ptr to each other — ref count never reaches 0.

Fix: Break the cycle with weak_ptr:

struct Parent;
struct Child {
  std::weak_ptr<Parent> parent_;  // GOOD: weak, not shared
};
struct Parent {
  std::shared_ptr<Child> child_;  // owns child

  // BAD version would have child holding shared_ptr<Parent>
  // → circular reference → memory leak
};

auto p = std::make_shared<Parent>();
auto c = std::make_shared<Child>();
c->parent_ = ____;  // p (assign weak_ptr — does not bump ref count)
p->child_ = c;

Failure 5: "undefined reference to vtable for MyClass"

Symptom: Linker error during colcon build.

Root Cause: Declared a pure virtual function but didn't implement the virtual destructor in the .cpp.

Fix: Define virtual destructor (even if empty) in exactly one .cpp file:

// sensor_base.cpp
#include "sensor_base.hpp"

// Even an empty destructor body anchors the vtable to this TU
SensorBase::~SensorBase() = ____;  // default

Failure Injection Drill

Exercise: Observe Dangling Pointer Crash

Deliberately capture a raw this in a delayed callback:

// dangling_this.cpp — intentional bug
#include <rclcpp/rclcpp.hpp>
#include <chrono>
using namespace std::chrono_literals;

class BadNode : public rclcpp::Node {
public:
  BadNode() : Node("bad_node") {
    // Capture raw 'this' — dangerous if node destroyed before timer fires
    timer_ = create_wall_timer(5s, [this]() {
      RCLCPP_INFO(get_logger(), "Timer fired");
    });
  }
  rclcpp::TimerBase::SharedPtr timer_;
};

int main(int argc, char** argv) {
  rclcpp::init(argc, argv);
  {
    auto node = std::make_shared<BadNode>();
    rclcpp::executors::SingleThreadedExecutor exec;
    exec.add_node(node);
    // Node destroyed when block exits, but timer still pending
  }  // node ref count → 0 → destroyed
  // Timer fires → accesses destroyed node → undefined behaviour!
  rclcpp::shutdown();
}

Observe:

  • Segfault or silent corruption
  • Address sanitizer output: heap-use-after-free
  • valgrind: Invalid read of size 8

Fix it:

// FIXED — shared_from_this() keeps node alive
timer_ = create_wall_timer(5s,
    [weak = std::weak_ptr<BadNode>(shared_from_this())]() {
      if (auto node = weak.lock()) {
        RCLCPP_INFO(node->get_logger(), "Timer fired safely");
      }
      // If node destroyed, weak.lock() returns nullptr — no crash
    });
Pro Tip:

Build with -fsanitize=address,undefined during development. In CMakeLists.txt: add_compile_options(-fsanitize=address -g) and add_link_options(-fsanitize=address). ASan catches use-after-free bugs immediately instead of causing mysterious crashes later.

Warning:

Never use std::bind(&MyClass::callback, this, ...) in a long-lived timer if there is any chance the node can be destroyed before the timer fires. Prefer lambda with shared_from_this() or weak_ptr capture.
