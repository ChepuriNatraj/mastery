Smart Pointers in ROS2 C++

Apply ownership models safely in async robotics code to avoid crashes and leaks.

Ownership Rules

auto unique_obj = std::make_unique<Controller>();
auto shared_obj = std::make_shared<Planner>();
std::weak_ptr<Planner> observer = shared_obj;

auto shared_copy = ____; // shared_obj

When to Use Which

  • unique_ptr: one owner only.
  • shared_ptr: asynchronous shared lifetime.
  • weak_ptr: non-owning references and parent back-links.

Failure Patterns

  • Cycle leak between two shared_ptr objects.
  • Dereference expired weak_ptr without lock().
  • Using moved-from unique_ptr.
