Parameters

Configure nodes dynamically without recompilation.

CPU-Light Required

What / Why / When

What

Parameters are runtime configuration values for nodes. Each parameter has a name, type, and value. Nodes declare parameters with defaults and validation constraints. ROS2 provides CLI and service interfaces to get/set parameters at runtime. Parameters can be loaded from YAML files or set via launch files.

Why

Hard-coding configuration requires recompilation for every tuning change. Parameters enable experimentation without rebuilding. They support environment-specific configs (sim vs hardware), A/B testing, and dynamic reconfigure for PID gains, thresholds, and behavior modes. Parameters are essential for production deployments across different robots.

When

Use parameters for any configurable value: gains, thresholds, timeouts, file paths, topic names. Declare parameters at node initialization with type constraints and ranges. For values that never change (constants), use class members instead. For calibration data, load from YAML. For runtime tuning, use parameter callbacks to react to changes.

10-Minute Drill

C++ Micro-Task

Declare and use parameters with validation:

#include "rclcpp/rclcpp.hpp"

class ParameterNode : public rclcpp::Node {
public:
  ParameterNode() : Node("parameter_node") {
    // Declare parameter with default value
    this->declare_parameter("max_speed", ____);
    this->declare_parameter("robot_name", ____);
    
    // Declare with range constraint
    auto descriptor = rcl_interfaces::msg::ParameterDescriptor();
    descriptor.description = "Sensor update rate in Hz";
    descriptor.integer_range.resize(1);
    descriptor.integer_range[0].from_value = 1;
    descriptor.integer_range[0].to_value = 100;
    this->declare_parameter("update_rate", 10, ____);
    
    // Get parameter values
    max_speed_ = this->get_parameter("____").as_double();
    robot_name_ = this->get_parameter("____").as_string();
    
    RCLCPP_INFO(this->get_logger(), "Max speed: %.2f, Robot: %s",
                max_speed_, robot_name_.c_str());
    
    // Set callback for dynamic changes
    param_callback_ = this->add_on_set_parameters_callback(
      std::bind(&ParameterNode::parameters_callback, this, std::placeholders::_1));
  }

private:
  rcl_interfaces::msg::SetParametersResult parameters_callback(
    const std::vector<rclcpp::Parameter>& parameters) {
    auto result = rcl_interfaces::msg::SetParametersResult();
    result.successful = true;
    
    for (const auto& param : parameters) {
      if (param.get_name() == "max_speed") {
        if (param.as_double() < 0.0 || param.as_double() > 10.0) {
          result.successful = false;
          result.reason = "max_speed must be between 0 and 10";
          return result;
        }
        max_speed_ = param.as_double();
        RCLCPP_INFO(this->get_logger(), "Updated max_speed to %.2f", max_speed_);
      }
    }
    return result;
  }

  double max_speed_;
  std::string robot_name_;
  rclcpp::Node::OnSetParametersCallbackHandle::SharedPtr param_callback_;
};

int main(int argc, char **argv) {
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<ParameterNode>());
  rclcpp::shutdown();
  return 0;
}

Python Micro-Task

Load parameters from YAML and react to changes:

import rclpy
from rclpy.node import Node
from rcl_interfaces.msg import ParameterDescriptor, SetParametersResult

class ParameterNode(Node):
    def __init__(self):
        super().__init__('parameter_node')
        
        # Declare with descriptor
        descriptor = ParameterDescriptor(
            description='Maximum linear velocity',
            read_only=False)
        self.declare_parameter('max_speed', ____, ____)
        self.declare_parameter('robot_name', ____)
        
        # Get values
        self.max_speed = self.get_parameter('____').value
        self.robot_name = self.get_parameter('____').value
        
        self.get_logger().info(f'Max speed: {self.max_speed}, Robot: {self.robot_name}')
        
        # Add callback
        self.add_on_set_parameters_callback(self.parameter_callback)
    
    def parameter_callback(self, params):
        result = SetParametersResult()
        result.successful = True
        
        for param in params:
            if param.name == 'max_speed':
                if param.value < 0.0 or param.value > 10.0:
                    result.successful = False
                    result.reason = 'max_speed must be 0-10'
                    return result
                self.max_speed = param.value
                self.get_logger().info(f'Updated max_speed: {self.max_speed}')
        
        return result

def main(args=None):
    rclpy.init(args=args)
    node = ParameterNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

Load from Launch File

from launch import LaunchDescription
from launch_ros.actions import Node
from ament_index_python.packages import get_package_share_directory
import os

def generate_launch_description():
    config = os.path.join(
        get_package_share_directory('my_pkg'),
        'config',
        'robot_params.yaml'
    )
    
    return LaunchDescription([
        Node(
            package='my_pkg',
            executable='parameter_node',
            name='parameter_node',
            parameters=[config]
        )
    ])

Verify

Command-Line Checks

# Terminal 1: Run node
ros2 run my_pkg parameter_node

# Terminal 2: List all parameters
ros2 param list
# Expected: /parameter_node max_speed, robot_name, update_rate, ...

# Get specific parameter
ros2 param get /parameter_node max_speed
# Expected: Double value: 2.5

# Set parameter at runtime
ros2 param set /parameter_node max_speed 3.5
# Expected: Set parameter successful (or rejection message if validation fails)

# Dump all parameters to file
ros2 param dump /parameter_node --output-dir ./config
# Creates yaml file with current values

# Load parameters from file
ros2 param load /parameter_node ./config/parameter_node.yaml

Describe Parameter

ros2 param describe /parameter_node update_rate
# Shows type, range, description, constraints
Pro Tip:

Use ros2 param dump to capture working configurations, then commit YAML files to version control for reproducibility.

Common Failures & Fix Patterns

Failure 1: Undeclared Parameter Access

Symptom: ParameterNotDeclaredException when calling get_parameter.

Root Cause: Trying to access parameter before declaring it or typo in parameter name.

Fix: Always declare before accessing:

# Wrong - will throw exception
val = self.get_parameter('speed').value  # Not declared yet

# Correct - declare first
self.declare_parameter('speed', 1.0)
val = self.get_parameter('speed').value

Failure 2: Type Mismatch

Symptom: Runtime error when calling as_int() on double parameter.

Root Cause: Parameter type doesn't match accessor method.

Fix: Use correct type accessor or check type first:

// Check type before accessing
auto param = this->get_parameter("value");
if (param.get_type() == rclcpp::ParameterType::PARAMETER_INTEGER) {
  int val = param.as_int();
} else {
  RCLCPP_ERROR(this->get_logger(), "Expected integer parameter");
}

Failure 3: Validation Ignored

Symptom: Invalid parameter values accepted, causing runtime errors.

Root Cause: No parameter callback to validate changes.

Fix: Always add validation callback:

rcl_interfaces::msg::SetParametersResult callback(
  const std::vector<rclcpp::Parameter>& params) {
  auto result = rcl_interfaces::msg::SetParametersResult();
  result.successful = true;
  
  for (const auto& p : params) {
    if (p.get_name() == "critical_param") {
      if (!validate(p.get_value())) {
        result.successful = false;
        result.reason = "Validation failed: out of range";
        return result;
      }
    }
  }
  return result;
}

Failure Injection Drill

Exercise: Range Validation

Create node with strict parameter constraints:

def parameter_callback(self, params):
    result = SetParametersResult()
    result.successful = True
    
    for param in params:
        if param.name == 'speed':
            if not (0.1 <= param.value <= 5.0):
                result.successful = False
                result.reason = f'Speed {param.value} out of range [0.1, 5.0]'
                self.get_logger().error(result.reason)
                return result
    
    return result

Detection: Attempt invalid set:

ros2 param set /node speed 10.0
# Expected: Setting parameter failed: Speed 10.0 out of range [0.1, 5.0]

Recovery: Provide clear error messages and valid range in rejection reason.

Warning:

Read-only parameters can't be changed after node starts. Use for immutable configs like robot ID or sensor calibration loaded at startup.
