Costmaps

Represent obstacles and free space for safe path planning and control.

Nav2 Perception

What / Why / When

What

Costmaps are 2D occupancy grids representing obstacle cost at each cell. Nav2 uses two costmaps: global (entire known map) and local (robot-centered rolling window). Each cell contains a cost from 0-254 (free to lethal), with 255 reserved for "unknown". Costmaps combine multiple layers: static map, obstacle inflation, sensor observations (lidar, depth cameras), and custom zones (keepout, speed limits). Updated continuously as robot moves and sensors detect obstacles. Used by planners to find collision-free paths and controllers to avoid dynamic obstacles.

Why

Raw sensor data and maps are insufficient for navigation - need unified representation of traversability. Costmaps merge heterogeneous inputs (map, laser, cameras) into single grid. Inflation layer grows obstacles by robot radius to simplify collision checking (if robot center avoids inflated obstacles, entire footprint is clear). Separate global/local costmaps enable efficient multi-resolution planning: global uses low-res entire map, local uses high-res immediate area. Layered architecture allows modular customization without changing core navigation code.

When

Costmaps are mandatory for Nav2 navigation. Global costmap is static for pre-mapped environments or updates slowly with SLAM. Local costmap updates at 5-20 Hz tracking sensor observations. Tune costmap size, resolution, and update frequency based on robot speed and environment complexity. Large costmaps (50m+) with high resolution (< 2cm) consume significant CPU/memory. Always visualize costmaps in RViz during tuning - misconfigurations cause navigation failures that are invisible without visualization.

Basic Costmap Configuration

Configure global and local costmaps:

# nav2_params.yaml
global_costmap:
  global_costmap:
    ros__parameters:
      # Frame and timing
      global_frame: ____  # map
      robot_base_frame: base_link
      update_frequency: 1.0  # Hz - slow for static map
      publish_frequency: 1.0
      
      # Dimensions
      width: 50  # meters
      height: 50
      resolution: ____  # 0.05 (5cm per cell)
      
      # Robot footprint (for collision checking)
      robot_radius: 0.22  # meters (if circular)
      # OR for polygonal footprint:
      # footprint: "[[0.3, 0.2], [0.3, -0.2], [-0.3, -0.2], [-0.3, 0.2]]"
      
      # Behavior
      rolling_window: ____  # false (fixed on map origin)
      track_unknown_space: true
      use_maximum: false
      
      # Plugins (layers)
      plugins: ["static_layer", "obstacle_layer", "inflation_layer"]
      
      # Individual layer configs below...

local_costmap:
  local_costmap:
    ros__parameters:
      global_frame: odom
      robot_base_frame: base_link
      update_frequency: ____  # 5.0 Hz (faster for dynamic obstacles)
      publish_frequency: 2.0
      
      width: 5  # meters (smaller than global)
      height: 5
      resolution: 0.05
      
      robot_radius: 0.22
      rolling_window: ____  # true (follows robot)
      track_unknown_space: false  # Assume clear
      
      plugins: ["voxel_layer", "inflation_layer"]

Static Layer (Map)

Load pre-built map into global costmap:

global_costmap:
  global_costmap:
    static_layer:
      plugin: "nav2_costmap_2d::StaticLayer"
      map_subscribe_transient_local: true
      enabled: true
      subscribe_to_updates: ____  # true
      map_topic: /map

Obstacle Layer (Sensor Data)

Add dynamic obstacles from laser scan:

global_costmap:
  global_costmap:
    obstacle_layer:
      plugin: "nav2_costmap_2d::ObstacleLayer"
      enabled: true
      
      # Observation sources
      observation_sources: scan
      
      scan:
        topic: ____  # /scan
        sensor_frame: laser_frame  # Must match TF
        data_type: "LaserScan"
        
        # Marking (add obstacles)
        marking: true
        max_obstacle_height: 2.0
        min_obstacle_height: 0.0
        obstacle_max_range: 2.5  # Only obstacles within 2.5m
        obstacle_min_range: ____  # 0.0
        
        # Clearing (remove stale obstacles)
        clearing: true
        raytrace_max_range: 3.0  # Clear up to 3m
        raytrace_min_range: 0.0
        
        # Filtering
        inf_is_valid: false

Inflation Layer (Safety Margin)

Expand obstacles by robot radius:

global_costmap:
  global_costmap:
    inflation_layer:
      plugin: "nav2_costmap_2d::InflationLayer"
      enabled: true
      
      # Inflation radius (robot radius + safety margin)
      inflation_radius: ____  # 0.55 (0.22 robot + 0.33 margin)
      
      # Cost scaling (exponential decay from obstacle)
      cost_scaling_factor: ____  # 3.0 (higher = steeper cost gradient)
      
      inflate_unknown: false
      inflate_around_unknown: true

Concept ↔ Code (Layer System)

Costmap Cell Values

Value Name Meaning
0 FREE_SPACE Completely clear, no obstacles
1-127 LOW_COST Inflated area, prefer to avoid
128-252 MEDIUM_COST Close to obstacles, high cost
253 INSCRIBED Robot inscribed radius boundary
254 LETHAL Obstacle detected, collision
255 UNKNOWN No information (unexplored)

Layer Update Order

// Conceptual costmap update cycle
void Costmap2D::updateMap() {
    // 1. Reset to default (typically unknown or free)
    resetMap();
    
    // 2. Apply each layer in order
    for (auto& layer : plugins_) {
        if (layer->isEnabled()) {
            // Static layer: copy map data
            // Obstacle layer: add sensor observations
            // Inflation layer: expand obstacles
            layer->updateCosts(master_grid_, min_i, min_j, max_i, max_j);
        }
    }
    
    // 3. Publish updated costmap
    publishCostmap();
}

Voxel Layer (3D Obstacles)

Alternative to obstacle layer with vertical clearing:

local_costmap:
  local_costmap:
    voxel_layer:
      plugin: "nav2_costmap_2d::VoxelLayer"
      enabled: true
      
      # 3D grid configuration
      origin_z: 0.0
      z_resolution: ____  # 0.2 (20cm per voxel)
      z_voxels: 16  # 16 * 0.2 = 3.2m max height
      
      # Mark/clear thresholds
      unknown_threshold: 15
      mark_threshold: 0
      
      # Observation sources (same as obstacle layer)
      observation_sources: scan
      scan:
        topic: /scan
        marking: true
        clearing: true
        max_obstacle_height: 2.0
        min_obstacle_height: ____  # 0.1 (ignore ground)

Custom Layer Plugin

// my_custom_layer.hpp
#include "nav2_costmap_2d/layer.hpp"

namespace my_costmap_plugins {

class MyCustomLayer : public nav2_costmap_2d::Layer {
public:
    virtual void onInitialize() override {
        // Setup subscriptions, parameters
        auto node = node_.lock();
        node->declare_parameter("my_param", 10.0);
    }
    
    virtual void updateBounds(
        double robot_x, double robot_y, double robot_yaw,
        double* min_x, double* min_y, double* max_x, double* max_y) override {
        // Define region that needs updating
        *min_x = robot_x - ____;  // 2.0
        *min_y = robot_y - 2.0;
        *max_x = robot_x + 2.0;
        *max_y = robot_y + ____;  // 2.0
    }
    
    virtual void updateCosts(
        nav2_costmap_2d::Costmap2D& master_grid,
        int min_i, int min_j, int max_i, int max_j) override {
        // Add costs to master_grid
        for (int j = min_j; j < max_j; j++) {
            for (int i = min_i; i < max_i; i++) {
                unsigned char cost = computeCustomCost(i, j);
                master_grid.setCost(i, j, cost);
            }
        }
    }
};

} // namespace

Inflation Tuning

Parameter Effect Tune When
inflation_radius Distance obstacles expand Robot size changes, add safety margin
cost_scaling_factor Cost gradient steepness Robot hugs walls (increase) or overly cautious (decrease)

Common Failures & Fix Patterns

Failure 1: Robot Treats Open Space as Obstacle

Symptom: Costmap shows lethal costs in known-clear areas, robot won't navigate through.

Root Cause: Sensor observations marking false obstacles (sensor noise, reflections).

Fix: Tune obstacle layer marking parameters:

obstacle_layer:
  scan:
    # Reduce marking range (ignore distant noise)
    obstacle_max_range: 3.0  # Was 5.0
    
    # Increase marking threshold (require multiple observations)
    # (For voxel layer only:)
    # mark_threshold: 2  # Need 2 hits to mark
    
    # Filter max range readings
    inf_is_valid: ____  # false
    
    # Clear aggressively
    raytrace_max_range: ____  # 4.0 (longer than marking range)

Failure 2: Obstacles Persist After Removal

Symptom: Move obstacle away but costmap still shows it as occupied.

Root Cause: Clearing disabled or insufficient raytrace range.

Fix: Enable clearing and verify raytrace:

obstacle_layer:
  scan:
    clearing: ____  # true (must be enabled)
    raytrace_max_range: 3.0  # At least as far as obstacle_max_range
    
    # For voxel layer, check thresholds
    unknown_threshold: 15
    mark_threshold: 0  # Easy to mark
    # Voxels decay over time when not observed

Failure 3: Robot Collides Despite Costmap

Symptom: Costmap shows obstacles but robot still hits them.

Root Cause: Incorrect robot footprint, inflation too small, or TF errors.

Fix: Verify footprint and inflation:

# Check actual robot dimensions
# Measure: length, width, turning radius

# Update footprint (add 5-10cm safety margin)
footprint: "[[0.35, 0.25], [0.35, -0.25], [-0.35, -0.25], [-0.35, 0.25]]"
# Was: [[0.3, 0.2], ...] but robot is actually bigger

# Increase inflation
inflation_radius: 0.6  # Was 0.4

# Verify TF alignment
ros2 run tf2_ros tf2_echo base_link laser_frame
# If transform is wrong, laser readings misaligned with robot center

Exercise: Disable Inflation and Observe

Remove inflation layer to see collision-free planning failure:

# Modify nav2_params.yaml
local_costmap:
  local_costmap:
    plugins: ["voxel_layer"]  # Removed "inflation_layer"
    
    # Restart Nav2
    # ros2 launch nav2_bringup navigation_launch.py

# Send navigation goal near obstacle

# Observe in RViz:
# - Planned path goes right next to obstacles (cost = 0 everywhere except obstacles)
# - Controller tries to follow path
# - Robot's corners/edges collide with obstacles
# - Without inflation, planner thinks robot center just needs to avoid obstacles
# - Forgets robot has physical size!

Detection: Check if inflation layer is enabled:

ros2 param get /local_costmap/local_costmap plugins
# Should include "inflation_layer"

# Visualize costmap in RViz
# Without inflation: Sharp black/white (lethal/free)
# With inflation: Gradient red→orange→yellow around obstacles

Recovery: Re-enable inflation:

plugins: ["voxel_layer", "____"]  # inflation_layer

# Set appropriate radius
inflation_radius: 0.55  # Robot radius (0.22) + margin (0.33)
Warning:

Costmap layers execute in order. Always place inflation_layer last - it needs final obstacle positions to compute gradients correctly.
