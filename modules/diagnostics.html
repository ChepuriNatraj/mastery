Diagnostics (Health Monitoring)

Monitor robot health with diagnostic updaters, aggregators, and alerts.

Health Check Monitoring

What / Why / When

What

ROS2 diagnostics framework monitors robot health through diagnostic_updater and diagnostic_aggregator. Diagnostic updaters publish status messages: OK, WARN, ERROR, STALE. Each diagnostic has key-value pairs (temperature: 45°C, voltage: 12.1V). Diagnostic aggregator collects status from multiple nodes, groups by category (sensors, motors, battery), and publishes aggregated health on /diagnostics_agg. RViz displays diagnostic status with color-coded tree. rqt_robot_monitor shows real-time alerts. Diagnostics enable proactive maintenance - detect degraded sensors before failure.

Why

Robots fail silently - wheel encoder stops, battery degrades, sensor saturates. Diagnostics surface these issues before mission failure. Aggregated view prevents alert fatigue - see "3 sensors degraded" instead of scrolling through 50 topics. Diagnostic logs enable root cause analysis - correlate navigation failure with low battery 2 minutes earlier. Threshold-based alerts (temp > 70°C) trigger cooling, human intervention. Fleet monitoring systems (10+ robots) rely on diagnostics for centralized health dashboards.

When

Add diagnostics to all hardware interfaces (cameras, lidars, motors, batteries). Publish status at 1-5 Hz. Thresholds: WARN at 80% of limit (temperature, voltage), ERROR at 95%. Use updater for per-node diagnostics (camera framerate, IMU noise). Use aggregator for system-level view (all sensors healthy?). Diagnostic messages are cheap - OK status is 100 bytes. Always log diagnostics to bag files for post-mission analysis. Integrate with external monitoring (Prometheus, Grafana) for fleet management.

10-Minute Drill

Diagnostic Status Levels

Level Value Meaning Action
OK 0 System healthy None - continue operation
WARN 1 Degraded but functional Log warning, schedule maintenance
ERROR 2 Critical failure Alert operator, abort mission if needed
STALE 3 No update received Check if node crashed, network issue

Basic Diagnostic Updater (C++)

// my_sensor_node.cpp
#include "rclcpp/rclcpp.hpp"
#include "diagnostic_updater/diagnostic_updater.hpp"
#include "diagnostic_updater/publisher.hpp"

class MySensorNode : public rclcpp::Node {
public:
    MySensorNode() : Node("my_sensor") {
        // Create diagnostic updater
        updater_.setHardwareID("sensor_v1");
        
        // Add diagnostic task
        updater_.add("Sensor Status", this, &MySensorNode::check_sensor);
        
        // Update diagnostics at 1 Hz
        timer_ = create_wall_timer(
            std::chrono::seconds(1),
            [this]() { updater_.force_update(); }
        );
    }

private:
    void check_sensor(diagnostic_updater::DiagnosticStatusWrapper &stat) {
        double temp = read_temperature();  // Your sensor code
        
        if (temp < 60.0) {
            stat.summary(____, "Temperature normal");  // diagnostic_msgs::msg::DiagnosticStatus::OK
            stat.add("Temperature", temp);
            stat.add("Unit", "Celsius");
        } else if (temp < 75.0) {
            stat.summary(diagnostic_msgs::msg::DiagnosticStatus::WARN, 
                         "Temperature high");
            stat.add("Temperature", temp);
        } else {
            stat.summary(diagnostic_msgs::msg::DiagnosticStatus::____, 
                         "Temperature critical");  // ERROR
            stat.add("Temperature", temp);
            stat.add("Action", "Shutdown recommended");
        }
    }
    
    diagnostic_updater::Updater updater_{this};
    rclcpp::TimerBase::SharedPtr timer_;
};

int main(int argc, char** argv) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared());
    rclcpp::shutdown();
    return 0;
}

Diagnostic Updater (Python)

#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from diagnostic_updater import Updater, DiagnosticStatusWrapper
from diagnostic_msgs.msg import DiagnosticStatus

class MySensorNode(Node):
    def __init__(self):
        super().__init__('my_sensor')
        
        # Create updater
        self.updater = Updater(self)
        self.updater.setHardwareID("sensor_v1")
        
        # Add diagnostic task
        self.updater.add("Sensor Status", self.check_sensor)
        
        # Timer to trigger updates
        self.timer = self.create_timer(1.0, self.update_diagnostics)
    
    def check_sensor(self, stat: DiagnosticStatusWrapper):
        temp = self.read_temperature()
        
        if temp < 60.0:
            stat.summary(____, "Temperature normal")  # DiagnosticStatus.OK
            stat.add("Temperature", f"{temp:.1f}")
            stat.add("Unit", "Celsius")
        elif temp < 75.0:
            stat.summary(DiagnosticStatus.WARN, "Temperature high")
            stat.add("Temperature", f"{temp:.1f}")
        else:
            stat.summary(DiagnosticStatus.ERROR, "Temperature critical")
            stat.add("Temperature", f"{temp:.1f}")
            stat.add("Action", "Shutdown recommended")
        
        return stat
    
    def update_diagnostics(self):
        self.updater.force_update()
    
    def read_temperature(self):
        # Your sensor reading code
        return 45.0

def main():
    rclpy.init()
    node = MySensorNode()
    rclpy.spin(node)
    rclpy.shutdown()

if __name__ == '__main__':
    main()

Aggregator Configuration

# diagnostic_aggregator.yaml
diagnostic_aggregator:
  ros__parameters:
    # Base path for aggregated diagnostics
    base_path: ''
    
    # Aggregation rules
    analyzers:
      # Group all sensors
      sensors:
        type: diagnostic_aggregator/AnalyzerGroup
        path: Sensors
        analyzers:
          camera:
            type: diagnostic_aggregator/GenericAnalyzer
            path: Camera
            contains: ['camera']  # Match any diagnostic with 'camera' in name
          lidar:
            type: diagnostic_aggregator/GenericAnalyzer
            path: Lidar
            contains: ['____']  # lidar, scan
      
      # Group all motors
      motors:
        type: diagnostic_aggregator/AnalyzerGroup
        path: Motors
        analyzers:
          wheels:
            type: diagnostic_aggregator/GenericAnalyzer
            path: Wheels
            contains: ['wheel', 'motor']
      
      # Battery monitoring
      power:
        type: diagnostic_aggregator/GenericAnalyzer
        path: Power
        contains: ['battery', 'voltage']

Frequency Diagnostic (Monitor Update Rate)

#include "diagnostic_updater/diagnostic_updater.hpp"
#include "diagnostic_updater/publisher.hpp"

class CameraNode : public rclcpp::Node {
public:
    CameraNode() : Node("camera_node") {
        pub_ = create_publisher("image", 10);
        
        // Monitor publisher frequency
        min_freq_ = 25.0;  // Expect 30 Hz, warn below 25
        max_freq_ = 35.0;  // Warn above 35
        
        freq_diag_ = std::make_unique(
            "image", updater_,
            diagnostic_updater::FrequencyStatusParam(&min_freq_, &max_freq_)
        );
        
        updater_.setHardwareID("camera_v2");
    }
    
    void publish_image(const sensor_msgs::msg::Image& img) {
        pub_->publish(img);
        freq_diag_->tick();  // Record publish event
    }

private:
    rclcpp::Publisher::SharedPtr pub_;
    diagnostic_updater::Updater updater_{this};
    std::unique_ptr freq_diag_;
    double min_freq_, max_freq_;
};

class BatteryMonitor {
public:
    void check_battery(diagnostic_updater::DiagnosticStatusWrapper &stat) {
        double voltage = read_battery_voltage();
        double current = read_battery_current();
        double soc = estimate_state_of_charge();  // 0-100%
        
        // Determine status
        if (soc > 20.0 && voltage > 11.5) {
            stat.summary(diagnostic_msgs::msg::DiagnosticStatus::OK, 
                         "Battery healthy");
        } else if (soc > 10.0) {
            stat.summary(diagnostic_msgs::msg::DiagnosticStatus::WARN,
                         "Battery low - return to dock soon");
        } else {
            stat.summary(diagnostic_msgs::msg::DiagnosticStatus::ERROR,
                         "Battery critical - return to dock immediately");
        }
        
        // Add detailed info
        stat.add("Voltage (V)", voltage);
        stat.add("Current (A)", ____);  // current
        stat.add("State of Charge (%)", soc);
        stat.add("Estimated Runtime (min)", estimate_runtime());
    }
};

Common Failures & Fix Patterns

Failure 1: Diagnostics Always STALE

Symptom: All diagnostics show STALE status despite node running.

Root Cause: Forgot to call updater.force_update() periodically.

Fix: Add timer to trigger updates:

# Missing periodic update
def __init__(self):
    self.updater = Updater(self)
    self.updater.add("Status", self.check_status)
    # No timer! Updater never called.

# Fixed version
def __init__(self):
    self.updater = Updater(self)
    self.updater.add("Status", self.check_status)
    
    # Add timer
    self.timer = self.create_timer(____, self.updater.force_update)  # 1.0 Hz

Failure 2: Aggregator Not Grouping Diagnostics

Symptom: Diagnostics appear flat, not grouped by category.

Root Cause: Analyzer path doesn't match diagnostic names.

Fix: Check name matching:

# Aggregator config
sensors:
  type: diagnostic_aggregator/GenericAnalyzer
  path: Sensors
  contains: ['camera']  # Matches 'camera_front', 'camera_back'

# Your diagnostic updater must have matching name
updater_.add("____", this, &check);  # camera_front (matches!)
# Debug: check diagnostic names
ros2 topic echo /diagnostics --once | grep "name:"
# Ensure names contain expected keywords

Failure 3: Diagnostic Overhead Too High

Symptom: Node slow, high CPU usage when diagnostics enabled.

Root Cause: Diagnostic update rate too high or expensive computation in diagnostic callback.

Fix: Reduce frequency and optimize callback:

// Bad: update at 100 Hz with expensive check
timer_ = create_wall_timer(
    std::chrono::milliseconds(10),  // 100 Hz - too fast!
    [this]() {
        run_expensive_test();  // Slow!
        updater_.force_update();
    }
);

// Good: update at 1 Hz with cached data
timer_ = create_wall_timer(
    std::chrono::seconds(____),  // 1 Hz
    [this]() { updater_.force_update(); }
);

void check_status(diagnostic_updater::DiagnosticStatusWrapper &stat) {
    // Use cached sensor data, don't read hardware here
    stat.add("Temperature", cached_temp_);
}

Failure Injection Drill

Exercise: Forget Hardware ID

Create diagnostic updater without setting hardware ID:

// Intentional error
MySensorNode() : Node("my_sensor") {
    // updater_.setHardwareID("sensor_v1");  // MISSING!
    updater_.add("Sensor Status", this, &MySensorNode::check_sensor);
}

Observe:

ros2 topic echo /diagnostics
# Output shows:
# hardware_id: ""  # Empty!

# In aggregator or rqt_robot_monitor:
# Diagnostic appears under "Unknown Hardware ID"

Detection: Check for empty hardware_id:

ros2 topic echo /diagnostics | grep "hardware_id: \"\""

Recovery:

MySensorNode() : Node("my_sensor") {
    updater_.setHardwareID("____");  // sensor_v1
    updater_.add("Sensor Status", this, &MySensorNode::check_sensor);
}
Warning:

Hardware ID helps identify which physical device has issues in multi-robot fleets. Always set unique IDs: robot_01_lidar_front, not just lidar.
