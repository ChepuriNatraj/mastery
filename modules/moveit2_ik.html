Inverse Kinematics: KDL vs TRAC-IK

Deeply understand IK behavior, solver trade-offs, and practical tuning so arm planning succeeds reliably in real robot workspaces.

What / Why / When

What

Inverse kinematics (IK) computes joint values for a desired end-effector pose. In MoveIt2, the kinematics plugin performs this step before trajectory generation. If IK fails, planning often fails immediately.

Why

Many “planner failures” are actually IK failures caused by singularities, unreachable targets, tight tolerances, or poor seed states. Solver choice directly affects mission reliability.

When

Use KDL for simple setups and quick defaults. Use TRAC-IK for higher success near joint limits and workspace boundaries, especially in pick-and-place and constrained manipulation.

KDL vs TRAC-IK Comparison

AspectKDLTRAC-IK
MethodIterative JacobianDual strategy (optimization + random restarts)
SpeedVery fastFast
Success near limitsModerateHigh
Joint limit handlingCan struggleStronger
Best useBaseline / simple reachabilityProduction manipulation reliability

Concept ↔ Code

geometry_msgs::msg::Pose target;
target.orientation.w = 1.0;
target.position.x = 0.45;
target.position.y = 0.00;
target.position.z = 0.35;

move_group.setPoseTarget(target);
move_group.setGoalPositionTolerance(0.005);
move_group.setGoalOrientationTolerance(0.01);

moveit::planning_interface::MoveGroupInterface::Plan plan;
bool ok = move_group.plan(plan) == moveit::core::MoveItErrorCode::SUCCESS;
if (ok) move_group.execute(plan);

// If failures persist near limits, increase planning attempts
move_group.setNumPlanningAttempts(____); // 5

Common Failures & Fix Patterns

  • Pose unreachable: move target inside workspace, validate frame and TCP link.
  • Intermittent IK: increase attempts and switch KDL → TRAC-IK.
  • Flipped wrist solutions: add orientation constraints and joint bounds.
  • Slow planning: reduce tolerances only as needed, not overly strict.

Verify

ros2 param get /move_group robot_description_kinematics
ros2 service call /compute_ik moveit_msgs/srv/GetPositionIK "{ik_request:{group_name:'arm'}}"
ros2 topic echo /rosout | grep -i -E "ik|kinematics|trac|kdl"
