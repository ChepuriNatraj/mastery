Lifecycle Nodes

Manage node states for safe startup and shutdown.

CPU-Light

What / Why / When

What

Lifecycle nodes implement a state machine with primary states: Unconfigured, Inactive, Active, and Finalized. Transitions between states are triggered externally via services. Each transition has callbacks (on_configure, on_activate, on_deactivate, on_cleanup, on_shutdown) for resource management. Lifecycle nodes enable coordinated system startup and graceful shutdown.

Why

Regular nodes start immediately and run until killed. This causes race conditions where nodes expect resources not yet available. Lifecycle nodes separate initialization (configure) from execution (activate). They enable safe reconfiguration without restart and coordinated multi-node shutdown. Nav2 uses lifecycle nodes extensively for controlled navigation stack management.

When

Use lifecycle nodes for hardware drivers that need initialization sequences, components requiring ordered startup, and nodes that must clean up resources before exit. Avoid lifecycle for simple stateless nodes. The lifecycle overhead is worthwhile when you need to reload configuration, swap algorithms, or ensure resource cleanup. Most Nav2 and hardware interface nodes use lifecycle.

C++ Micro-Task

Create a lifecycle node with state callbacks:

#include "rclcpp/rclcpp.hpp"
#include "rclcpp_lifecycle/lifecycle_node.hpp"
#include "lifecycle_msgs/msg/transition.hpp"

using CallbackReturn = rclcpp_lifecycle::node_interfaces::LifecycleNodeInterface::CallbackReturn;

class LifecycleDemo : public rclcpp_lifecycle::LifecycleNode {
public:
  LifecycleDemo() : LifecycleNode("lifecycle_demo") {
    RCLCPP_INFO(this->get_logger(), "Lifecycle node created");
  }

  CallbackReturn on_configure(const rclcpp_lifecycle::State& previous_state) {
    RCLCPP_INFO(this->get_logger(), "Configuring from %s", previous_state.label().c_str());
    
    // Load parameters, initialize resources
    this->declare_parameter("update_rate", ____);
    update_rate_ = this->get_parameter("____").as_int();
    
    // Create publishers (not active yet)
    pub_ = this->create_publisher<std_msgs::msg::String>("output", 10);
    
    // Return SUCCESS or FAILURE
    return CallbackReturn::____;
  }

  CallbackReturn on_activate(const rclcpp_lifecycle::State& previous_state) {
    RCLCPP_INFO(this->get_logger(), "Activating from %s", previous_state.label().c_str());
    
    // Activate publishers and start timers
    LifecycleNode::on_activate(previous_state);
    
    timer_ = this->create_wall_timer(
      std::chrono::seconds(1),
      std::bind(&LifecycleDemo::timer_callback, this));
    
    return CallbackReturn::SUCCESS;
  }

  CallbackReturn on_deactivate(const rclcpp_lifecycle::State& previous_state) {
    RCLCPP_INFO(this->get_logger(), "Deactivating from %s", previous_state.label().c_str());
    
    // Stop publishing, cancel timers
    LifecycleNode::on_deactivate(previous_state);
    timer_->cancel();
    
    return CallbackReturn::SUCCESS;
  }

  CallbackReturn on_cleanup(const rclcpp_lifecycle::State& previous_state) {
    RCLCPP_INFO(this->get_logger(), "Cleaning up from %s", previous_state.label().c_str());
    
    // Release resources
    timer_.reset();
    pub_.reset();
    
    return CallbackReturn::SUCCESS;
  }

  CallbackReturn on_shutdown(const rclcpp_lifecycle::State& previous_state) {
    RCLCPP_INFO(this->get_logger(), "Shutting down from %s", previous_state.label().c_str());
    
    // Emergency cleanup
    timer_.reset();
    pub_.reset();
    
    return CallbackReturn::SUCCESS;
  }

private:
  void timer_callback() {
    if (this->get_current_state().id() == 
        lifecycle_msgs::msg::State::PRIMARY_STATE_ACTIVE) {
      auto msg = std_msgs::msg::String();
      msg.data = "Active: " + std::to_string(count_++);
      pub_->publish(msg);
      RCLCPP_INFO(this->get_logger(), "Publishing: %s", msg.data.c_str());
    }
  }

  std::shared_ptr<rclcpp_lifecycle::LifecyclePublisher<std_msgs::msg::String>> pub_;
  rclcpp::TimerBase::SharedPtr timer_;
  int update_rate_;
  int count_ = 0;
};

int main(int argc, char **argv) {
  rclcpp::init(argc, argv);
  auto node = std::make_shared<LifecycleDemo>();
  rclcpp::spin(node->get_node_base_interface());
  rclcpp::shutdown();
  return 0;
}

Python Micro-Task

Create matching lifecycle manager to control transitions:

import rclpy
from rclpy.node import Node
from lifecycle_msgs.srv import ChangeState, GetState
from lifecycle_msgs.msg import Transition

class LifecycleManager(Node):
    def __init__(self):
        super().__init__('lifecycle_manager')
        
        # Create service clients for target node
        self.change_state_client = self.create_client(
            ChangeState,
            '/lifecycle_demo/____')  # change_state
        
        self.get_state_client = self.create_client(
            GetState,
            '/lifecycle_demo/get_state')
        
        while not self.change_state_client.wait_for_service(timeout_sec=1.0):
            self.get_logger().info('Waiting for lifecycle services...')
    
    def get_state(self):
        request = GetState.Request()
        future = self.get_state_client.call_async(request)
        rclpy.spin_until_future_complete(self, future)
        return future.result()
    
    def change_state(self, transition_id):
        request = ChangeState.Request()
        request.transition.id = ____
        
        future = self.change_state_client.call_async(request)
        rclpy.spin_until_future_complete(self, future)
        
        result = future.result()
        if result.success:
            self.get_logger().info(f'Transition successful')
        else:
            self.get_logger().error(f'Transition failed')
        return result.success
    
    def configure(self):
        return self.change_state(Transition.____)  # TRANSITION_CONFIGURE
    
    def activate(self):
        return self.change_state(Transition.TRANSITION_ACTIVATE)
    
    def deactivate(self):
        return self.change_state(Transition.TRANSITION_DEACTIVATE)
    
    def cleanup(self):
        return self.change_state(Transition.TRANSITION_CLEANUP)
    
    def shutdown(self):
        return self.change_state(Transition.TRANSITION_UNCONFIGURED_SHUTDOWN)

def main(args=None):
    rclpy.init(args=args)
    manager = LifecycleManager()
    
    # Orchestrate lifecycle transitions
    manager.get_logger().info('Configuring node...')
    manager.configure()
    
    import time
    time.sleep(1)
    
    manager.get_logger().info('Activating node...')
    manager.activate()
    
    time.sleep(5)
    
    manager.get_logger().info('Deactivating node...')
    manager.deactivate()
    
    manager.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

Lifecycle State Machine

                     ┌──────────────┐
                     │ Unconfigured │ (initial state)
                     └──────┬───────┘
                            │ configure()
                            ▼
                     ┌──────────────┐
                     │   Inactive   │ (configured but not running)
                     └──┬───────┬───┘
               activate()│       │ cleanup()
                         ▼       ▼
                  ┌──────────────┐  ┌──────────────┐
                  │    Active    │  │ Unconfigured │
                  └──────┬───────┘  └──────────────┘
                         │ deactivate()
                         ▼
                  ┌──────────────┐
                  │   Inactive   │
                  └──────┬───────┘
                         │ shutdown()
                         ▼
                  ┌──────────────┐
                  │  Finalized   │ (terminal state)
                  └──────────────┘

State Transition IDs

Transition From State To State ID
configure Unconfigured Inactive 1
cleanup Inactive Unconfigured 2
activate Inactive Active 3
deactivate Active Inactive 4
shutdown (unconfigured) Unconfigured Finalized 5
shutdown (inactive) Inactive Finalized 6
shutdown (active) Active Finalized 7

Launch Lifecycle Node with Manager

from launch import LaunchDescription
from launch_ros.actions import Node, LifecycleNode
from launch.actions import RegisterEventHandler
from launch_ros.event_handlers import OnStateTransition
from launch_ros.events.lifecycle import ChangeState
from lifecycle_msgs.msg import Transition

def generate_launch_description():
    # Launch lifecycle node
    lifecycle_node = LifecycleNode(
        package='my_pkg',
        executable='lifecycle_demo',
        name='lifecycle_demo',
        namespace='',
        output='screen'
    )
    
    # Automatically configure on startup
    configure_event = RegisterEventHandler(
        OnStateTransition(
            target_lifecycle_node=lifecycle_node,
            goal_state='inactive',
            entities=[
                # Trigger activation after configuration
                ChangeState(
                    lifecycle_node_name='lifecycle_demo',
                    transition_id=Transition.TRANSITION_ACTIVATE
                )
            ]
        )
    )
    
    # Initial configuration trigger
    initial_configure = RegisterEventHandler(
        OnStateTransition(
            target_lifecycle_node=lifecycle_node,
            start_state='configuring',
            goal_state='inactive',
            entities=[
                # Log transition
            ]
        )
    )
    
    return LaunchDescription([
        lifecycle_node,
        configure_event,
        # Trigger initial configure
        ChangeState(
            lifecycle_node_name='lifecycle_demo',
            transition_id=Transition.TRANSITION_CONFIGURE
        )
    ])

Common Failures & Fix Patterns

Failure 1: Transition Callback Returns FAILURE

Symptom: Node remains in previous state after transition request, logs show callback failure.

Root Cause: Configuration error in on_configure (e.g., parameter not found, file missing).

Fix: Add error handling and return descriptive failures:

CallbackReturn on_configure(const rclcpp_lifecycle::State& previous_state) {
  try {
    // Load config file
    std::string config_path = this->get_parameter("config_file").as_string();
    if (!load_config(config_path)) {
      RCLCPP_ERROR(this->get_logger(), "Failed to load config: %s", config_path.c_str());
      return CallbackReturn::FAILURE;
    }
    return CallbackReturn::SUCCESS;
  } catch (const std::exception& e) {
    RCLCPP_ERROR(this->get_logger(), "Exception in configure: %s", e.what());
    return CallbackReturn::ERROR;
  }
}

Failure 2: Publishing from Inactive State

Symptom: No messages published even though node is configured.

Root Cause: Lifecycle publishers don't send messages until activated.

Fix: Use LifecyclePublisher and activate it:

// Create LifecyclePublisher in on_configure
pub_ = this->create_publisher<std_msgs::msg::String>("topic", 10);

// Must call on_activate to enable publishing
CallbackReturn on_activate(const rclcpp_lifecycle::State& previous_state) {
  LifecycleNode::on_activate(previous_state);  // Activates all lifecycle publishers
  return CallbackReturn::SUCCESS;
}

// Publishing only works in Active state
void publish() {
  if (this->get_current_state().id() == lifecycle_msgs::msg::State::PRIMARY_STATE_ACTIVE) {
    pub_->publish(msg);
  }
}

Failure 3: Resource Leak on Transition Failure

Symptom: Resources (files, connections) not released when configure fails.

Root Cause: No cleanup in on_cleanup or exception before resource release.

Fix: Use RAII or explicit cleanup on all exit paths:

CallbackReturn on_configure(const rclcpp_lifecycle::State&) {
  try {
    hardware_interface_ = std::make_unique<HardwareInterface>();
    if (!hardware_interface_->connect()) {
      hardware_interface_.reset();  // Clean up on failure
      return CallbackReturn::FAILURE;
    }
    return CallbackReturn::SUCCESS;
  } catch (...) {
    hardware_interface_.reset();
    throw;
  }
}

CallbackReturn on_cleanup(const rclcpp_lifecycle::State&) {
  hardware_interface_.reset();  // Release in cleanup
  return CallbackReturn::SUCCESS;
}

Failure Injection Drill

Exercise: Handle Configuration Failure Gracefully

Create lifecycle node that fails configuration based on parameter:

from rclpy.lifecycle import Node as LifecycleNode, TransitionCallbackReturn, State

class FlakyLifecycleNode(LifecycleNode):
    def on_configure(self, state: State):
        self.declare_parameter('fail_configure', False)
        
        if self.get_parameter('fail_configure').value:
            self.get_logger().error('Configuration intentionally failed')
            return TransitionCallbackReturn.FAILURE
        
        self.get_logger().info('Configuration successful')
        return TransitionCallbackReturn.SUCCESS

Detection: Test with parameter override:

# Should succeed
ros2 run my_pkg flaky_node
ros2 lifecycle set /flaky_node configure
# Expected: Transitioning successful

# Should fail
ros2 run my_pkg flaky_node --ros-args -p fail_configure:=true
ros2 lifecycle set /flaky_node configure
# Expected: Transitioning failed, node remains unconfigured

Recovery: Detect failure and retry with corrected parameters or fallback configuration.

Warning:

Lifecycle transition callbacks block the node. Avoid long-running operations in callbacks. Use separate threads for slow initialization and signal completion via state check.
