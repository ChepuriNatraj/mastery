Path Planners

Compute collision-free global paths from start to goal using A*, Dijkstra, and hybrid search.

Nav2 Planning

What / Why / When

What

Path planners compute obstacle-free routes from robot's current pose to goal using global costmap. Nav2 provides multiple planner plugins: NavFn (Dijkstra/A*), Smac Planner (hybrid-A* with kinematic constraints), and Theta* (any-angle). Planners return nav_msgs/Path - sequence of PoseStamped waypoints. Planning happens at low frequency (0.1-1 Hz) since global map changes slowly. Results are kinematically feasible (Smac) or geometric (NavFn) paths that controllers then track. Support features: tolerance goals, alternative paths, and planning timeouts.

Why

Controllers alone can't navigate complex environments - need global guidance to avoid local minima (dead ends, traps). Planners use full map knowledge to find optimal routes considering obstacle layout. Different planners trade speed vs quality: NavFn is fast but ignores robot kinematics, Smac is slower but generates drivable paths for car-like robots. Replanning enables dynamic obstacle avoidance - when path is blocked, planner computes new route. Efficient planners are critical for responsive navigation in large environments.

When

Use NavFn for differential drive robots in simple environments. Use Smac Hybrid-A* for ackermann (car-like) steering or when paths must be kinematically feasible. Use Theta* when smooth, direct paths are preferred over grid-aligned. Planner choice affects computational load significantly: NavFn ~10ms, Smac ~100-500ms, Theta* ~20-50ms. Always set planning timeout and handle failure cases. Test planners offline in RViz before deployment - visualize paths to verify quality.

Set up planner with multiple plugin options:

# nav2_params.yaml
planner_server:
  ros__parameters:
    use_sim_time: true
    
    # Planner plugins
    expected_planner_frequency: ____  # 1.0 Hz
    planner_plugins: ["GridBased", "SmacHybrid"]
    
    # NavFn Planner (Dijkstra/A*)
    GridBased:
      plugin: "nav2_navfn_planner/NavfnPlanner"
      tolerance: 0.5  # Goal tolerance in meters
      use_astar: ____  # true (A* faster than Dijkstra)
      allow_unknown: false  # Don't plan through unknown space
      use_final_approach_orientation: false
    
    # Smac Hybrid-A* Planner
    SmacHybrid:
      plugin: "nav2_smac_planner/SmacPlannerHybrid"
      
      # Search parameters
      tolerance: 0.5
      downsample_costmap: false
      downsampling_factor: 1
      
      # Kinematic limits
      minimum_turning_radius: ____  # 0.4 meters (for car-like robots)
      angle_quantization_bins: 72  # 5-degree resolution
      
      # Heuristic
      cost_travel_multiplier: 2.0
      reverse_penalty: 2.0  # Prefer forward motion
      change_penalty: 0.1  # Penalize direction changes
      non_straight_penalty: 1.2
      
      # Analytic expansion (shortcut to goal)
      analytic_expansion_ratio: 3.5
      analytic_expansion_max_length: 3.0
      
      # Smoothing
      smoother:
        max_iterations: 1000
        w_smooth: ____  # 0.3
        w_data: 0.2
        tolerance: 1e-10

Concept ↔ Code (Algorithms)

NavFn Planner (A* Search)

# Conceptual A* algorithm
import heapq

def a_star(start, goal, costmap):
    # Priority queue: (f_score, pose)
    open_set = [(0, start)]
    came_from = {}
    
    g_score = {start: 0}
    f_score = {start: heuristic(start, goal)}
    
    while open_set:
        current = heapq.heappop(open_set)[1]
        
        if current == goal:
            return reconstruct_path(came_from, current)
        
        for neighbor in get_neighbors(current, costmap):
            # Cost = g + edge_cost
            tentative_g = g_score[current] + ____  # edge_cost(current, neighbor)
            
            if tentative_g < g_score.get(neighbor, float('inf')):
                came_from[neighbor] = current
                g_score[neighbor] = tentative_g
                f_score[neighbor] = tentative_g + heuristic(neighbor, ____)  # goal
                
                if neighbor not in [p[1] for p in open_set]:
                    heapq.heappush(open_set, (f_score[neighbor], neighbor))
    
    return None  # No path found

def heuristic(pose, goal):
    # Euclidean distance
    dx = goal.x - pose.x
    dy = goal.y - pose.y
    return (dx**2 + dy**2) ** 0.5

Planner Comparison

Planner Algorithm Speed Quality Use Case
NavfnPlanner A* / Dijkstra Fast (10ms) Grid-aligned, ignores kinematics Differential drive, simple environments
SmacPlannerHybrid Hybrid-A* Slow (100-500ms) Kinematically feasible, smooth Ackermann steering, parking
SmacPlanner2D A* with SE2 Medium (50ms) Orientation-aware, grid-based Diff drive with heading constraints
ThetaStarPlanner Theta* Medium (20-50ms) Any-angle, shorter paths Open spaces, prefer direct routes

Smac Hybrid-A* Explanation

Hybrid-A* Search Space:
- States: (x, y, theta) - position + heading
- Expansions: Simulate vehicle motion model
  → Forward motion
  → Forward + left steer
  → Forward + right steer
  → Reverse motion (with penalty)
  
Advantages:
- Paths respect turning radius constraints
- No post-processing smoothing needed (Smac has built-in)
- Analytic expansion: Try direct curve to goal

Disadvantages:
- 3D search space (x, y, theta) vs 2D (x, y)
- Slower: Must check collision along curves
- Needs accurate vehicle model parameters

#include "nav2_core/global_planner.hpp"

namespace my_planner {

class MyPlanner : public nav2_core::GlobalPlanner {
public:
    void configure(
        const rclcpp_lifecycle::LifecycleNode::WeakPtr & parent,
        std::string name,
        std::shared_ptr tf,
        std::shared_ptr costmap_ros) override {
        
        node_ = parent.lock();
        name_ = name;
        tf_ = tf;

        // Declare parameters

    }
    
    nav_msgs::msg::Path createPlan(
        const geometry_msgs::msg::PoseStamped & start,
        const geometry_msgs::msg::PoseStamped & goal) override {
        
        nav_msgs::msg::Path path;
        path.header.frame_id = ____;  // "map"
        path.header.stamp = node_->now();
        
        // Your planning algorithm here
        // 1. Convert start/goal to costmap coordinates
        // 2. Run search (A*, RRT, etc.)
        // 3. Convert result to poses
        
        auto poses = compute_path_algorithm(start, goal);
        path.poses = poses;
        
        return path;
    }
};

} // namespace

Benchmark Planners

# benchmark_planners.py
import rclpy
from rclpy.node import Node
from nav2_msgs.srv import ComputePathToPose
from geometry_msgs.msg import PoseStamped
import time

class PlannerBenchmark(Node):
    def __init__(self):
        super().__init__('planner_benchmark')
        self.client = self.create_service_client(
            ComputePathToPose, '/compute_path_to_pose')
    
    def benchmark(self, planner_id, start, goal):
        request = ComputePathToPose.Request()
        request.start.header.frame_id = 'map'
        request.goal.header.frame_id = 'map'
        request.start.pose.position.x = start[0]
        request.start.pose.position.y = start[1]
        request.goal.pose.position.x = goal[0]
        request.goal.pose.position.y = goal[1]
        request.planner_id = planner_id
        
        start_time = time.time()
        future = self.client.call_async(request)
        rclpy.spin_until_future_complete(self, future)
        elapsed = time.time() - start_time
        
        result = future.result()
        path_length = len(result.path.poses)
        
        self.get_logger().info(
            f'{planner_id}: {elapsed*1000:.1f}ms, {path_length} waypoints')

# Test:
# node.benchmark('GridBased', (0, 0), (10, 10))
# node.benchmark('SmacHybrid', (0, 0), (10, 10))
Pro Tip:

Use NavFn for initial path, then switch to Smac for refinement. In BT: ComputePathToPose with GridBased (fast), if controller fails, replan with SmacHybrid (better quality).

Common Failures & Fix Patterns

Failure 1: Planning Timeout

Symptom: "Planner timed out" error, no path returned even though path exists.

Root Cause: Complex environment, planner too slow, or costmap too large/fine-resolution.

Fix: Increase timeout or optimize costmap:

# Increase planning time
SmacHybrid:
  max_planning_time: 10.0  # Was 5.0

# OR reduce search space
global_costmap:
  global_costmap:
    resolution: 0.1  # Was 0.05 (coarser = faster)
    
# OR downsample for planning
SmacHybrid:
  downsample_costmap: ____  # true
  downsampling_factor: 2  # Use half resolution

Failure 2: Path Too Close to Obstacles

Symptom: Planner returns path but controller can't follow (robot too close to walls).

Root Cause: Planner uses costmap without considering controller's safety margins.

Fix: Increase costmap inflation for planning:

# In global_costmap inflation layer
inflation_layer:
  inflation_radius: 0.8  # Was 0.5 (increase for planning)
  cost_scaling_factor: 3.0
  
# Planner will now avoid inflated areas
# Alternatively, use separate planning costmap with larger inflation

Failure 3: Smac Returns Infeasible Path

Symptom: Smac Hybrid-A* path has turns sharper than robot can execute.

Root Cause: Incorrect minimum_turning_radius parameter.

Fix: Measure and set actual turning radius:

# Measure robot's turning radius:
# 1. Drive robot in circle at max steering
# 2. Measure diameter, divide by 2

# Update config
SmacHybrid:
  minimum_turning_radius: ____  # 1.5 (measured value)
  
# Verify: Send goal requiring tight turn
# Path should now have feasible curvature

# Look for "Planning failed" or "Timeout" status

# Check service response
ros2 service call /compute_path_to_pose ... 
# Response has empty path.poses array when planning fails

Recovery: Behavior tree handles failure:

<!-- BT recovery pattern -->
<RecoveryNode number_of_retries="3">
  <ComputePathToPose .../>
  
  <!-- On failure: -->
  <SequenceStar>
    <ClearCostmap/>  <!-- Maybe obstacle is stale -->
    <Wait duration="2"/>  <!-- Let costmap update -->
  </SequenceStar>
</RecoveryNode>
Warning:

Smac planners cache analytic expansions for performance. If robot parameters change at runtime, restart planner_server to clear cache, or paths may be infeasible.
