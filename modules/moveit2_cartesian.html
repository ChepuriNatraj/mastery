Cartesian Path Planning

Plan straight-line tool motion safely with waypoint interpolation, fraction checks, and collision-aware execution.

What / Why / When

computeCartesianPath() interpolates end-effector movement in workspace coordinates. Use it for insertion, approach/retreat, polishing, and tasks requiring linear tool travel. Use OMPL for long global moves and Cartesian planning for precise local segments.

10-Minute Drill

std::vector<geometry_msgs::msg::Pose> waypoints;
auto start = move_group.getCurrentPose().pose;

auto p1 = start; p1.position.z -= 0.10; waypoints.push_back(p1);
auto p2 = p1;    p2.position.x += 0.05; waypoints.push_back(p2);

moveit_msgs::msg::RobotTrajectory traj;
const double eef_step = 0.005;
const double jump_threshold = ____; // 0.0

double fraction = move_group.computeCartesianPath(waypoints, eef_step, jump_threshold, traj);
if (fraction >= 0.95) {
  moveit::planning_interface::MoveGroupInterface::Plan p; p.trajectory_ = traj;
  move_group.execute(p);
}

Common Failures

  • Low fraction: reduce eef_step, split path.
  • Unexpected joint flips: enable jump threshold and smooth orientation changes.
  • Collisions: add planning scene objects and maintain clearance.
